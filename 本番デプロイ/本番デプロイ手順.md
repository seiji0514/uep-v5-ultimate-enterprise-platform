# 本番デプロイ手順

## このフォルダについて

`本番デプロイ/` は本番デプロイ用の起動スクリプトをまとめたフォルダです。

### UEP v5.0 本体

| ファイル | 用途 |
|----------|------|
| start-production-all.bat | バックエンド＋フロントエンドを一括起動 |
| start-backend-production.bat | バックエンドのみ起動（ポート8080） |
| start-frontend-production.bat | フロントエンドのみ起動（ポート3000） |

### 産業統合プラットフォーム

| ファイル | 用途 |
|----------|------|
| start-industry-backend-production.bat | バックエンドのみ起動（ポート9010） |
| start-industry-frontend-production.bat | フロントエンドのみ起動（ポート3010） |

### 企業横断オペレーション基盤（EOH）

| ファイル | 用途 |
|----------|------|
| start-eoh-backend-production.bat | バックエンドのみ起動（ポート9020） |
| start-eoh-frontend-production.bat | フロントエンドのみ起動（ポート3020） |

※ UEP 一括起動時はプロジェクトルートの `start-*-production-launcher.bat` を経由して別ウィンドウを開きます。

## 本番デプロイ時の手順

### 1. ドメインを設定

```bash
python scripts/set-production-domain.py your-domain.com
```

例: `uep.mycompany.co.jp` の場合
```bash
python scripts/set-production-domain.py uep.mycompany.co.jp
```

→ API: https://api.uep.mycompany.co.jp  
→ フロント: https://uep.mycompany.co.jp, https://app.uep.mycompany.co.jp

### 2. Sentry を設定（推奨）

1. [sentry.io](https://sentry.io) でプロジェクト作成
2. DSN を取得
3. `frontend/.env.production` の `REACT_APP_SENTRY_DSN=` に設定

### 3. 起動

#### 方法1: 一括起動

```
本番デプロイ\start-production-all.bat
```

→ バックエンド（別ウィンドウ）＋フロントエンド（ビルド＋起動）

#### 方法2: 個別起動

```
REM ターミナル1: バックエンド
本番デプロイ\start-backend-production.bat

REM ターミナル2: フロントエンド（ビルド＋起動）
本番デプロイ\start-frontend-production.bat
```

→ http://localhost:3000 で本番ビルドを確認

## 産業統合プラットフォーム・EOH の起動

### 産業統合プラットフォーム（ポート9010/3010）

```
REM ターミナル1: バックエンド
本番デプロイ\start-industry-backend-production.bat

REM ターミナル2: フロントエンド
本番デプロイ\start-industry-frontend-production.bat
```

初回: `industry_unified_platform/frontend/.env.production` に `REACT_APP_INDUSTRY_API_URL=http://localhost:9010` を設定

### 企業横断オペレーション基盤（ポート9020/3020）

```
REM ターミナル1: バックエンド
本番デプロイ\start-eoh-backend-production.bat

REM ターミナル2: フロントエンド
本番デプロイ\start-eoh-frontend-production.bat
```

事前準備: `enterprise_operations_hub/backend/.env` を作成し、`EOH_SECRET_KEY` を設定（`openssl rand -hex 32` で生成）。詳細は `docs/PRODUCTION_DEPLOYMENT.md` 参照。

## デプロイ先への配置

- **フロント**: `frontend/build` を静的ホスト（Nginx, S3, Vercel 等）に配置
- **バックエンド**: Gunicorn/Uvicorn で起動、リバースプロキシ経由で HTTPS 化
