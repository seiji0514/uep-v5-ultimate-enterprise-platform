# システム改善の余地（4プラットフォーム）

**作成日**: 2026年3月16日  
**対象**: UEP v5.0、統合セキュリティ・防衛プラットフォーム、産業統合プラットフォーム、企業横断オペレーション基盤

**実装完了**: 2026年3月16日  
- Phase A: 統合セキュリティ・産業統合・EOH の a11y（aria-label・EmptyState・OfflineBanner）を実装完了
- Phase B: 3プラットフォーム E2E テスト（Playwright）、フォームバリデーション（ログイン・タスク追加）を実装完了
- Phase C: フォーカスリング（2px outline）、UEP 連携（統合セキュリティ・防衛をダッシュボード・メニュー・検索に追加、SSO対応）を実装完了

---

## サマリ

| プラットフォーム | 改善余地 | 優先度の高い項目 |
|------------------|----------|------------------|
| **UEP v5.0** | あり | 残タスク（Phase B/C）、パフォーマンス、E2E拡充 |
| **統合セキュリティ・防衛** | あり | UEP連携、a11y、EmptyState、E2E |
| **産業統合** | あり | UEP連携、a11y、EmptyState、シミュレーション拡張 |
| **企業横断オペレーション基盤（EOH）** | あり | UEP連携強化、a11y、オフライン対応 |

## 1. UEP v5.0

### 改善案（[UEP_v5.0_システム改善案.md](./UEP_v5.0_システム改善案.md) より）

| カテゴリ | 改善項目 | 現状 | 改善案 |
|----------|----------|------|--------|
| **UX/UI** | レスポンシブ | 一部対応 | モバイル・タブレットのメニュー最適化 |
| **a11y** | スクリーンリーダー | 未検証 | セマンティックHTML・見出し階層の整理 |
| **a11y** | コントラスト | 要確認 | WCAG 2.1 AA準拠 |
| **パフォーマンス** | バンドルサイズ | 要計測 | Tree shaking・未使用インポート削除 |
| **パフォーマンス** | 画像最適化 | 該当少 | WebP・遅延読み込み（将来） |
| **堅牢性** | トークン期限切れ | リフレッシュ対応済 | 細部の動作確認 |
| **機能** | 多言語対応 | 拡張済 | 英語切り替えの完成度向上 |
| **開発者体験** | E2Eテスト | 拡充済 | 6モジュール・現場AI以外のフロー追加 |
| **バックエンド** | レート制限 | グローバル | エンドポイント別の細分化 |

### 即時対応可能（小規模）

- 残存ページの aria-label 確認
- ローディング状態の統一確認
- フォームバリデーションの他ページへの展開

## 2. 統合セキュリティ・防衛プラットフォーム

### 現状

- 個別システム（UEP 起動時には含まれない）
- ポート 9001（API）、3001（フロント）
- イベント・インシデント・リスク・IDS/IPS・EDR・SIEM・脅威インテリジェンス・コンプライアンス

### 改善の余地

| カテゴリ | 改善項目 | 改善案 |
|----------|----------|--------|
| **UEP連携** | ダッシュボード統合 | UEP ダッシュボードからシームレスに遷移（同一タブ or 埋め込み） |
| **a11y** | aria-label・キーボード操作 | 主要ボタン・タブ・フォームに aria-label、Tab 操作対応 |
| **UX/UI** | EmptyState | 空データ時の統一表示 |
| **UX/UI** | ローディング | CircularProgress に aria-label、スケルトン |
| **堅牢性** | フォームバリデーション | 検索・フィルタのリアルタイムバリデーション |
| **開発者体験** | E2Eテスト | Playwright で主要フロー（イベント一覧・インシデント・脅威インテリ） |
| **レスポンシブ** | モバイル | drawer 幅・パディングの最適化 |

---

## 3. 産業統合プラットフォーム

### 現状

- 製造・医療・金融・セキュリティの4ドメインを1つに統合
- ポート 9010（API）、3010（フロント）
- 8ドメイン（インフラ・DX・コンプライアンス・サプライチェーン含む）

### 改善の余地

| カテゴリ | 改善項目 | 改善案 |
|----------|----------|--------|
| **UEP連携** | ダッシュボード統合 | UEP からの遷移を同一タブで統一（現状維持確認） |
| **a11y** | aria-label・キーボード操作 | タブ・チップ・ドロップダウンに aria-label |
| **UX/UI** | EmptyState | 各ドメインの空データ表示を統一 |
| **UX/UI** | ローディング | シミュレーション更新時のローディング表示 |
| **シミュレーション** | データ拡張 | よりリアルなサンプルデータ、更新間隔の柔軟化 |
| **開発者体験** | E2Eテスト | タブ切り替え・要対応サマリ・シミュレーションモード |
| **レスポンシブ** | モバイル | 8ドメインタブのモバイル表示最適化 |

---

## 4. 企業横断オペレーション基盤（EOH）

### 現状

- 観測・タスク・リスクの一元管理
- ポート 9020（API）、3020（フロント）
- 認証・RBAC・DB永続化・通知・CSV/Excelエクスポート・PWA対応

### 改善の余地

| カテゴリ | 改善項目 | 改善案 |
|----------|----------|--------|
| **UEP連携** | トークンSSO | UEP からの遷移時のトークン引き継ぎの安定化 |
| **a11y** | aria-label・キーボード操作 | 観測・タスク・リスクの一覧・フォームに aria-label |
| **UX/UI** | EmptyState | 観測・タスク・リスクの空データ表示 |
| **堅牢性** | オフライン対応 | Service Worker・OfflineBanner（UEP と同様） |
| **パフォーマンス** | キャッシュ | React Query の staleTime・gcTime 最適化 |
| **開発者体験** | E2Eテスト | 観測・タスク追加・リスク登録の主要フロー |
| **機能** | アラート | 閾値超過・期限間近の通知の強化 |
