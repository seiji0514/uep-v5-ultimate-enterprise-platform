## 📋 プロジェクト概要

**プロジェクト名**: 次世代エンタープライズ統合プラットフォーム v5.0 (UEP v5.0)  
**期間**: 2026年1月（約1ヶ月）  
**役割**: フルスタック開発（設計・実装・テスト）

### プロジェクトの目的

企業が最重視するIT戦略課題と需要が高いシステムを統合したエンタープライズプラットフォームの構築。

### 統合システム（7つのコアシステム）

1. **MLOps基盤システム** - MLパイプライン、モデル管理、実験追跡
2. **生成AIシステム** - LLM統合、RAG、CoT推論
3. **監視・オブザーバビリティシステム** - メトリクス収集、SLO管理、ログ管理
4. **統合セキュリティコマンドセンター** - セキュリティ監視、インシデント対応、リスク分析
5. **クラウドインフラシステム** - インフラ管理、IaC、オーケストレーション
6. **統合開発・運用プラットフォーム（IDOP）** - CI/CD、DevOps管理
7. **AI支援開発システム** - コード生成、テスト自動化、コードレビュー

## 🛠️ 使用技術

### バックエンド
- **言語**: Python 3.11+
- **フレームワーク**: FastAPI
- **認証・認可**: JWT, OAuth2/OIDC, RBAC/ABAC
- **API**: RESTful API, Swagger UI

### フロントエンド
- **言語**: TypeScript
- **フレームワーク**: React 18+
- **UIライブラリ**: Material-UI (MUI)
- **ルーティング**: React Router
- **HTTPクライアント**: Axios

### インフラ・ツール
- **API Gateway**: Kong, Envoy
- **データレイク**: MinIO
- **イベントストリーミング**: Apache Kafka
- **監視**: Prometheus, Grafana, ELK Stack
- **セキュリティ**: Zero Trust, mTLS, Vault
- **コンテナ**: Docker, Docker Compose
- **IaC**: Terraform

## 🚀 デモンストレーション手順

### 前提条件
- Windows 10/11
- Python 3.11以上
- Node.js 18以上

### 起動方法

#### 1. バックエンドの起動
```cmd
start-backend.bat
```
- 起動確認: `http://localhost:8000/health`
- APIドキュメント: `http://localhost:8000/docs`

#### 2. フロントエンドの起動（別のコマンドプロンプト）
```cmd
start-frontend.bat
```
- 起動確認: `http://localhost:3000`

#### 3. ログイン
- **URL**: `http://localhost:3000`
- **ユーザー名**: `kaho0525`
- **パスワード**: `kaho052514`

#### 4. 機能確認
以下の機能を確認できます：
- **ダッシュボード**: システム概要とナビゲーション
- **MLOps**: パイプライン、モデル、実験の管理
- **生成AI**: テキスト生成、RAGクエリ、CoT推論
- **セキュリティコマンドセンター**: セキュリティイベント、インシデント管理
- **クラウドインフラ**: リソース管理、IaCテンプレート
- **IDOP**: CI/CDパイプライン、アプリケーション管理
- **AI支援開発**: コード生成、コードレビュー、ドキュメント生成
- **設定**: プロフィール、セキュリティ、通知設定

## 💡 解決した主要な課題

### 1. Windows環境での実行
**課題**: 初期実装はWSL/Docker前提だった  
**解決**: Windowsネイティブのバッチファイル（`.bat`）を作成し、直接実行可能に  
**成果**: デモンストレーション環境の構築時間を大幅に短縮

### 2. 認証システムの実装
**課題**: OAuth2/OIDC、JWT、RBAC/ABACの統合  
**解決**: JWT認証とRBACデコレータを実装  
**成果**: セキュアな認証・認可システムの構築

### 3. フロントエンドの実装
**課題**: React + TypeScript + Material-UIでのエンタープライズレベルのUI実装  
**解決**: Material-UIを使用したモダンなUI、React Routerによるルーティング、認証コンテキストによる状態管理  
**成果**: 統一されたUI/UXと保守性の高いコードベース

### 4. APIエラーハンドリング
**課題**: 422バリデーションエラー、ネットワークエラーなどの適切な処理  
**解決**: 詳細なエラーメッセージの表示、バリデーションエラーの詳細表示、FastAPIデコレータの修正  
**成果**: ユーザーフレンドリーなエラー表示とデバッグの容易さ

### 5. セキュリティ設定
**課題**: セキュリティヘッダーとCSPの適切な設定  
**解決**: Swagger UI用のCSP緩和、セキュリティヘッダーの適切な設定  
**成果**: セキュアなAPIドキュメントの提供
