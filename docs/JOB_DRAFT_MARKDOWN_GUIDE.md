## 📋 マークダウン記法について

### フィードバックの内容

**「マークダウン記法が利用できますので、是非ご活用ください。」**

転職ドラフトスカウトのレジュメ記載欄では、マークダウン記法が利用可能です。

---

## ✅ マークダウン記法の使い方

### 基本的なマークダウン記法

#### 1. 見出し（`#`）

**マークダウン記法**:
```markdown
# 見出し1
## 見出し2
### 見出し3
```

**記載例**:
```markdown
# プロジェクト経験概要
## チーム情報
### 開発・実装内容A
```

**表示結果**: 
- `#` は見出しとして表示される（通常は非表示）
- 見出しが太字や大きいフォントで表示される

---

#### 2. 太字（`**`）

**マークダウン記法**:
```markdown
**太字**
```

**記載例**:
```markdown
**開発体制**: 個人開発（1人）
**自身が行ったこと**:
```

**表示結果**: 
- `**` は非表示
- テキストが太字で表示される

---

#### 3. リスト（`-`）

**マークダウン記法**:
```markdown
- 項目1
- 項目2
- 項目3
```

**記載例**:
```markdown
- プロジェクトマネージャー（PM）
- システムアーキテクト
- バックエンドエンジニア
```

**表示結果**: 
- `-` は箇条書きの記号として表示される
- リスト形式で表示される

---

#### 4. コードブロック（`` ` ``）

**マークダウン記法**:
```markdown
`コード`
```

**記載例**:
```markdown
`FastAPI`、`React`、`TypeScript`
```

**表示結果**: 
- `` ` `` は非表示
- テキストがコード形式で表示される（背景色が変わる）

---

#### 5. コードブロック（```）

**マークダウン記法**:
```markdown
```
コードブロック
```
```

**記載例**:
```markdown
```
FastAPI
React
TypeScript
```
```

**表示結果**: 
- ``` は非表示
- コードブロックとして表示される（背景色が変わる）

## 💡 マークダウン記法の使い方のポイント

### ポイント1: 見出しで構造化

**`#` を使用して見出しを作成**

- ✅ `#` で大見出しを作成
- ✅ `##` で中見出しを作成
- ✅ `###` で小見出しを作成

**例**:
```markdown
# プロジェクト経験概要
## チーム情報
### 開発・実装内容A
```

---

### ポイント2: 太字で強調

**`**` を使用して重要な部分を強調**

- ✅ `**太字**` で重要な部分を強調
- ✅ プロジェクト名、役割、技術名などを強調

**例**:
```markdown
**開発体制**: 個人開発（1人）
**自身が行ったこと**:
- **FastAPI**: バックエンドAPIを実装
```

---

### ポイント3: リストで整理

**`-` を使用してリストを作成**

- ✅ `-` で箇条書きを作成
- ✅ 役割、技術スタック、成果などをリスト化

**例**:
```markdown
- プロジェクトマネージャー（PM）
- システムアーキテクト
- バックエンドエンジニア
```

---

### ポイント4: コード形式で技術名を表示

**`` ` `` を使用して技術名をコード形式で表示**

- ✅ `` `技術名` `` で技術名をコード形式で表示
- ✅ 技術名を強調し、読みやすくする

**例**:
```markdown
`FastAPI`、`React`、`TypeScript`を使用して実装
```

---

### ポイント5: コードブロックで技術スタックを表示

**``` を使用してコードブロックを作成**

- ✅ ``` でコードブロックを作成
- ✅ 技術スタック、コード例などを表示

**例**:
```markdown
```
FastAPI
React
TypeScript
```
```

## 📋 実際の記載方法

### 方法1: マークダウン記法をそのまま記載（推奨）

**転職ドラフトスカウトのレジュメ記載欄に、マークダウン記法をそのまま記載**

- ✅ `#` で見出しを作成
- ✅ `**` で太字を作成
- ✅ `-` でリストを作成
- ✅ `` ` `` でコード形式を作成

**例**:
```markdown
# プロジェクト経験概要

**プロジェクト名**: 次世代エンタープライズ統合プラットフォーム v5.0 (UEP v5.0)

**期間**: 2024年8月 〜 2026年1月（個人開発・就活中）

**役割**: 
- プロジェクトマネージャー（PM）
- システムアーキテクト
- バックエンドエンジニア
```

**表示結果**: 
- マークダウン記法が自動的に変換され、見やすく表示される
- 見出し、太字、リストなどが適切に表示される

---

### 方法2: マークダウン記法を使わない（非推奨）

**マークダウン記法を使わずに、通常のテキストで記載**

- ⚠️ 可読性が低下する可能性がある
- ⚠️ フィードバックで「マークダウン記法を活用してください」と推奨されている

**例**:
```
プロジェクト経験概要

プロジェクト名: 次世代エンタープライズ統合プラットフォーム v5.0 (UEP v5.0)

期間: 2024年8月 〜 2026年1月（個人開発・就活中）

役割: 
プロジェクトマネージャー（PM）
システムアーキテクト
バックエンドエンジニア
```

**表示結果**: 
- マークダウン記法がないため、見出しや太字が表示されない
- 可読性が低下する可能性がある
