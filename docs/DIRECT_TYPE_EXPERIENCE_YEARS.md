## 💡 まとめ

**「1年未満」ではなく、職務経歴全体を考慮して経験年数を記載することを推奨します。**

#### 経験年数の計算方法

1. **職務経歴全体を考慮**: 各スキル・技術の経験年数は、職務経歴全体を考慮して計算
2. **具体的内容を記載**: 「個人開発」だけでなく、具体的な職務内容、期間、実績を記載
3. **実績を強調**: 100システム以上の実装実績、エンタープライズレベルの統合プラットフォーム開発などの実績を強調

#### 記載例

```
【経験年数】
5年以上

【具体的内容】
AI技術研究開発（2025年6月～現在、約8ヶ月、個人開発、100システム以上の実装実績）+ フルリモート業務（2020年3月～2024年8月、約4年5ヶ月、Webアクセシビリティ検査、Webサイト開発）
```

## 📝 最終的な推奨

### 推奨：職務経歴全体を考慮して経験年数を記載 ⭐⭐⭐⭐⭐

**「1年未満」ではなく、職務経歴全体を考慮して経験年数を記載することを強く推奨します。**

#### 理由

- ✅ **正確な経験年数**: 職務経歴全体を考慮することで、正確な経験年数を記載できる
- ✅ **実績の強調**: 100システム以上の実装実績、エンタープライズレベルの統合プラットフォーム開発などの実績を強調できる
- ✅ **採用担当者の理解**: 採用担当者が理解しやすい

#### 記載方法

```
【経験年数】
5年以上（または適切な年数）

【具体的内容】
AI技術研究開発（2025年6月～現在、約8ヶ月、個人開発、100システム以上の実装実績）+ フルリモート業務（2020年3月～2024年8月、約4年5ヶ月、Webアクセシビリティ検査、Webサイト開発）
```

#### 注意点

- ⚠️ **「個人開発」だけでは不十分**: 「個人開発」だけでなく、具体的な職務内容、期間、実績を記載
- ⚠️ **職務経歴全体を考慮**: 各スキル・技術の経験年数は、職務経歴全体を考慮して計算
