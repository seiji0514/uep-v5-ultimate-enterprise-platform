デモで使うAI用語集（聞かれたときの説明用）

面談で「〇〇とは？」と聞かれたとき、簡潔に説明できるようピックアップ。

RAG（ラグ、Retrieval-Augmented Generation）

意味: 検索拡張生成。外部知識を検索してLLM（エルエルエム）の入力に加え、根拠のある回答を生成。ハルシネーション軽減。

デモでの説明: 「生成AI（セイセイエーアイ）のRAG（ラグ）タブで、ドキュメント検索とLLM（エルエルエム）による回答生成の連携をご確認いただけます。」

CoT（コット、Chain-of-Thought）推論

意味: 思考の連鎖。段階的に推論してから結論を出す手法。複雑な問題で精度向上。

デモでの説明: 「CoT（コット）推論をプロンプトに組み込み、論理的な回答を生成するように実装しています。」

MLOps / LLMOps（エムエルオプス / エルエルエムオプス）

意味: MLOps（エムエルオプス）=ML（エムエル）モデルの運用管理。LLMOps（エルエルエムオプス）=LLM（エルエルエム）の運用管理（プロンプト管理、監視、A/Bテスト等）。

デモでの説明: 「MLOps（エムエルオプス）タブでパイプライン・モデル・実験管理、Prometheus（プロメテウス）/Grafana（グラファナ）で稼働率99.995%を監視しています。」

推論AI・モデルルーティング（スイロンエーアイ・モデルルーティング）

意味: タスク難度に応じて軽量モデル／高性能モデルを振り分ける。コストと品質のバランス。

デモでの説明: 「インダストリーページの推論AI（スイロンエーアイ）で、タスクに応じたモデルルーティングを実装しています。」

MCP / A2A（エムシーピー / エーツーエー）

意味: MCP（エムシーピー）=AI（エーアイ）が外部ツールにアクセスするプロトコル。A2A（エーツーエー）=エージェント間連携。

デモでの説明: 「インダストリーのLevel 4で、MCP（エムシーピー）/A2A（エーツーエー）によるエージェント間連携を統合しています。」

レイテンシ（レイテンシ、Latency）

意味: 応答時間。推論レイテンシ=AI（エーアイ）の回答速度。10ms未満はリアルタイム性を意味。

デモでの説明: 「推論レイテンシを10ms未満に最適化し、ユーザー体験を確保しています。」

ベクトルデータベース（ベクトルデータベース、ChromaDB等）

意味: テキストをベクトル化して類似検索するDB（ディービー）。RAG（ラグ）の「関連ドキュメント検索」に使用。

デモでの説明: 「ChromaDB（クローマディービー）をベクトルDB（ディービー）として使い、RAG（ラグ）で関連ドキュメントを検索しています。」

API / REST API / OpenAPI（エーピーアイ / レストエーピーアイ / オープンエーピーアイ）

意味: API（エーピーアイ）=ソフト間の連携インターフェース。REST（レスト）=HTTP（エイチティーティーピー）ベースの設計。OpenAPI（オープンエーピーアイ）=API仕様の標準。

デモでの説明: 「Swagger UI（スワッガーユーアイ）でOpenAPI（オープンエーピーアイ）ドキュメントを自動生成し、RESTful API（レストフルエーピーアイ）を実装しています。」

マイクロサービス / イベント駆動（マイクロサービス / イベントクドウ）

意味: マイクロサービス=小さなサービスに分割。イベント駆動=イベントで疎結合に連携。

デモでの説明: 「Level 1～5をマイクロサービスで独立実装し、イベント駆動で疎結合に連携しています。」

稼働率99.995%（カドウリツ99.995%）

意味: 可用性の指標。99.995%=月間ダウンタイム約2分。Prometheus（プロメテウス）/Grafana（グラファナ）で監視。

デモでの説明: 「監視基盤で稼働率99.995%を測定し、エンタープライズレベルの品質を達成しています。」

インクルーシブ雇用AI（インクルーシブコヨウエーアイ）

意味: 障害者雇用マッチング、アクセシビリティ特化AI、当事者視点UX評価を統合したプラットフォーム。身体障害者としての当事者視点を活かしたシステム。

デモでの説明: 「サイドメニューのインクルーシブ雇用AIで、求人マッチング・アクセシビリティAIチャット・企業サイトのUX評価・AIエージェントをご確認いただけます。障害者雇用とアクセシビリティに特化した実装です。」

デモの進め方

デモは面談の流れに合わせて実施する。事前に環境を起動し、手順書を手元に用意しておく。

  • 実績・プロジェクトを聞かれたとき 「実際に動作しているシステムがあります。画面共有でデモンストレーションできます」と伝え、了承されたら画面共有でデモを行う。
  • 技術的な深掘りをされたとき UEP（ユーイーピー）v5.0 を起動し、Swagger UI（スワッガーユーアイ）とフロントのダッシュボードを中心に説明する。
  • AI用語の意味を聞かれたとき 上記「デモで使うAI用語集」を参照し、簡潔に説明してから実装例を補足。
  • 時間が足りない／話題が別のとき 無理にデモはせず、口頭で概要（Level 1～5 統合、技術スタック、品質指標など）を説明する。
  • 「デモを見せてください」と言われたとき すぐに画面共有に切り替え、「面談当日の起動手順」以降に従ってデモを行う。
  • 事前にしておくこと start-backend.bat、フロントエンド（cd frontend → npm start）で起動できる状態にしておく。本手順書を開いておく。Zoom（ズーム）/Meet（ミート）の画面共有を確認しておく。

推奨デモフロー（約10分）

よくある質問

Q1: RAG（ラグ）とは？ 検索拡張生成。外部知識を検索してLLM（エルエルエム）の入力に加え、根拠のある回答を生成。ハルシネーション軽減。上記「デモで使うAI用語集」参照。

Q2: CoT（コット）とは？ 思考の連鎖。段階的に推論してから結論を出す手法。上記「デモで使うAI用語集」参照。

Q3: なぜWindows（ウィンドウズ）環境？ デモ環境としてWindowsデスクトップPCを想定。WSL（ダブリューエスエル）やDocker Desktop（ドッカーデスクトップ）が不要で簡単に起動できるため。

Q4: 認証情報の変更は？ backend/auth/routes.py の _init_demo_users で変更可能。現在は kaho0525 / kaho052514。

Q5: 本番環境での使用は？ 開発・デモ用。本番ではセキュリティ・DB（ディービー）・環境変数・ログの見直しが必要。

Q6: パフォーマンスは？ 開発環境向け。本番ではDB（ディービー）最適化、キャッシング、ロードバランシング、CDN（シーディーエヌ）を推奨。

Q7: レベル5グローバルエンタープライズ対応とは？ グローバル展開を想定した設計。マルチリージョン、高可用性、コンプライアンス、災害復旧まで考慮。Level 1（7コア）から Level 5（グローバル）まで段階的に拡張できる構成。

Q8: インクルーシブ雇用AIとは？ 障害者雇用マッチング、アクセシビリティ特化AI、当事者視点UX評価を統合したプラットフォーム。身体障害者としての当事者視点を活かした実装。上記「デモで使うAI用語集」参照。

「グローバル展開を想定した設計で、マルチリージョン、高可用性、コンプライアンス、災害復旧まで考慮しています。Level 1の7コアから、Level 5のグローバルエンタープライズまで段階的に拡張できる構成です。」

ITエンジニア実務経験・チーム開発について

面談で「実務経験は？」「チーム開発は？」と聞かれたときの回答例。

ITエンジニア実務経験について

Q: 未経験ですか？実務経験はありますか？

回答のポイント: 未経験ではありません。以下のIT関連経験があります。

  • KDDIエボルバ（11ヶ月）: ヘルプデスク業務。セキュリティのためのパスワード再設定、サーバー処理を担当。IT・通信業界での技術業務。
  • AI技術研究開発（8ヶ月）: 個人での開発・設計・実装。100システム以上を設計・実装。Python、FastAPI、React、TypeScript、Docker、Kubernetes等を使用。
  • JCI Teleworkers' Network（2年2ヶ月）: Webアクセシビリティ検査。Web技術関連の検査業務を完全リモートで実施。
「未経験ではありません。KDDIエボルバでヘルプデスク・サーバー処理を11ヶ月、その後AI技術研究開発で約8ヶ月、個人開発として100システム以上を設計・実装してきました。JCIではWebアクセシビリティ検査を2年2ヶ月担当しています。」

チーム開発について

Q: チーム開発の経験はありますか？個人開発ばかりでは？

回答のポイント: チーム開発経験は浅いが、リモートでの協働ツールには慣れている。

  • 個人開発が中心。チームでのコードレビュー・ペアプログラミングの経験は浅い。
  • 以前のフルリモート勤務（Earth Technology 1年5ヶ月、JCI 2年2ヶ月等）でSlack・Teams・Google Meetを日常的に使用。
  • 非同期コミュニケーションとオンライン会議には慣れており、リモートでのチーム連携ツールは問題なく使える。
  • コードレビューは経験が浅いため、指摘を素直に受け止め、質問しながら学ぶ姿勢で臨む。

トラブル時の対応

バックエンドが起動しない: ポート8000が使用中の場合は8001で起動する。python（パイソン）--version 確認 → venv（ベンブ）再作成 → それでもダメなら Swagger UI（スワッガーユーアイ）のスクリーンショットで説明

フロントエンドが起動しない: node（ノード）--version 確認 → node_modules（ノードモジュール）削除 → npm（エヌピーエム）install → それでもダメなら Swagger UI（スワッガーユーアイ）でAPI（エーピーアイ）実演

ログインできない: バックエンド起動確認 → 認証情報確認 → それでもダメなら Swagger UI（スワッガーユーアイ）でAPI（エーピーアイ）実演

画面共有ができない: スクリーンショット共有 または コードを直接共有して説明

  4. インクルーシブ雇用AI（障害者雇用マッチング・アクセシビリティ特化AI・当事者視点UX評価）による差別化
  5. 統合プラットフォームとしての完成度（Level 1～5 グローバルエンタープライズ対応、2025-2026最新4技術統合、ビジネス領域12種）
