# サイバー対策 レベル4 本番運用の具体的な手順

**作成日**: 2026年3月2日  
**対象**: UEP v5.0 サイバー対策基盤（Suricata, Wazuh, OpenSearch, MISP）

---

## 1. 前提条件

### 1.1 インフラ要件

| 項目 | 要件 |
|------|------|
| **OS** | Linux (Ubuntu 22.04 LTS 推奨) |
| **CPU** | 8コア以上（推奨 16コア） |
| **メモリ** | 32GB 以上（推奨 64GB） |
| **ストレージ** | 500GB 以上 SSD（ログ保持用） |
| **ネットワーク** | パケットミラーリング or SPAN ポート |

### 1.2 ネットワーク構成

```
[本番ネットワーク]
       │
       │ パケットミラー or SPAN
       ▼
[Suricata] ──→ eve.json ──→ Filebeat ──→ OpenSearch
       │
       └──→ UEP Backend (POST /cyber-defense/suricata/alerts)

[監視対象サーバー] ──→ Wazuh Agent ──→ Wazuh Manager ──→ UEP Backend
```

## 2. 本番デプロイ手順

### Step 1: ネットワーク準備

```bash
# 1. パケットキャプチャ用インターフェースの確認
ip link show

# 2. Suricata 用に SPAN ポートからトラフィックを受信する設定
# （スイッチ側でミラーポート設定が必要）

# 3. ファイアウォールで必要なポートを開放
# Suricata: なし（パケットキャプチャのみ）
# Wazuh: 1514, 1515 (Agent), 5500 (API)
# OpenSearch: 9200
# UEP Backend: 8000
```

### Step 2: Docker ネットワーク作成

```bash
# メインの docker-compose で uep-network を作成
docker network create uep-network 2>/dev/null || true

# または既存の compose で起動
docker-compose up -d
```

### Step 3: Suricata 本番起動

```bash
# 1. ルールセットの取得（Emerging Threats 無料版）
docker run --rm -v $(pwd)/infrastructure/cyber-defense/suricata/rules:/rules \
  jasonish/suricata -T

# 2. 本番用 suricata.yaml の編集
# - interface: パケットキャプチャ用 NIC 名
# - HOME_NET: 自社ネットワーク CIDR
# - rules: ルールファイルパス

# 3. 起動（network_mode: host でパケットキャプチャ）
docker run -d --name suricata \
  --network host \
  --cap-add=NET_RAW --cap-add=NET_ADMIN \
  -v $(pwd)/infrastructure/cyber-defense/suricata/suricata.yaml:/etc/suricata/suricata.yaml \
  -v $(pwd)/infrastructure/cyber-defense/suricata/rules:/var/lib/suricata/rules \
  -v /var/log/suricata:/var/log/suricata \
  jasonish/suricata -c /etc/suricata/suricata.yaml -i eth0
```

### Step 4: eve.json の UEP 連携（Filebeat または Python スクリプト）

```bash
# 方法A: Filebeat で OpenSearch に送信し、UEP は OpenSearch から取得
# 方法B: Python で eve.json を tail し、UEP API に POST

# 方法B の簡易スクリプト例（cron または systemd で定期実行）
# scripts/suricata_to_uep.py を参照
```

### Step 5: Wazuh Manager 起動

```bash
# 1. OpenSearch が起動していることを確認
curl -k http://localhost:9200

# 2. Wazuh Manager 起動
docker-compose -f docker-compose.yml \
  -f infrastructure/cyber-defense/docker-compose.cyber-defense.yml \
  --profile cyber-defense up -d wazuh-manager

# 3. エージェント登録用のキー取得
docker exec -it uep-wazuh-manager /var/ossec/bin/agent-auth -m wazuh-manager
```

### Step 6: 監視対象サーバーへの Wazuh Agent インストール

```bash
# Ubuntu/Debian の場合
curl -sL https://packages.wazuh.com/4.7/wazuh-agent-4.7.0-1.deb -o wazuh-agent.deb
sudo dpkg -i wazuh-agent.deb

# Manager の IP を設定
sudo sed -i 's/MANAGER_IP=.*/MANAGER_IP=<Wazuh-Manager-IP>/' /var/ossec/etc/ossec.conf

# 起動
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent
```

### Step 7: UEP Backend の本番設定

```bash
# backend/.env
ENVIRONMENT=production
DEBUG=false
SECRET_KEY=<本番用64文字以上>
DATABASE_URL=postgresql://postgres:xxx@postgres:5432/uep_db
REDIS_URL=redis://redis:6379/0

# セキュリティ関連（オプション）
SURICATA_EVE_PATH=/var/log/suricata/eve.json
WAZUH_API_URL=https://wazuh-manager:5500
WAZUH_API_USER=wazuh
WAZUH_API_PASSWORD=<設定したパスワード>
MISP_URL=https://misp.example.com
MISP_API_KEY=<MISP APIキー>
```

### Step 8: UEP 起動

```bash
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

## 4. インシデント対応フロー

```
1. アラート検知（UEP / Suricata / Wazuh）
   ↓
2. 重大度判定（Critical / High）
   ↓
3. 初動対応
   - 影響範囲の特定
   - 関連ログの保全
   - 関係者への連絡
   ↓
4. 封じ込め
   - 該当ホストのネットワーク分離
   - 該当アカウントの無効化
   ↓
5. 復旧
   - マルウェア除去
   - パッチ適用
   - システム復旧
   ↓
6. 事後対応
   - インシデントレポート作成
   - 再発防止策の検討
   - UEP インシデントに記録
```

## 6. トラブルシューティング

### Suricata がアラートを検知しない

- パケットキャプチャ用 NIC が正しいか確認
- `network_mode: host` で起動しているか確認
- ルールが有効か確認: `suricata -T`

### Wazuh Agent が Manager に接続できない

- ファイアウォールで 1514, 1515 が開放されているか確認
- Manager の IP が正しいか確認
- `sudo systemctl status wazuh-agent` でエラー確認

### UEP にアラートが表示されない

- 外部ツールから UEP API への POST が成功しているか確認
- 認証トークンが有効か確認
- バックエンドログでエラー確認
