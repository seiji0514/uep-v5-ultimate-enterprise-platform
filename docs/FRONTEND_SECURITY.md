## セキュリティ脆弱性の確認と修正

### 脆弱性の確認

```cmd
cd frontend
npm audit
```

### 自動修正

修正可能な脆弱性を自動修正：

```cmd
npm audit fix
```

### 強制修正（破壊的変更を含む）

すべての脆弱性を修正（破壊的変更の可能性あり）：

```cmd
npm audit fix --force
```

## セキュリティベストプラクティス

### 1. 依存関係の管理

- 定期的に`npm audit`を実行
- 不要な依存関係を削除
- 最新の安定版を使用

### 2. 環境変数

- `.env`ファイルに機密情報を含めない
- `.gitignore`に`.env`を追加（既に追加済み）

### 3. API通信

- HTTPSを使用（本番環境）
- 認証トークンの安全な保存
- CORS設定の確認

### 4. 認証

- JWTトークンの適切な管理
- トークンの有効期限チェック
- ログアウト時のトークン削除
