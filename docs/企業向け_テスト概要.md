# 企業向け テスト概要

**更新**: 2026年2月

---

## 現状のテスト構成

### バックエンド（backend/tests/）

| テスト種別 | ファイル | 内容 |
|------------|----------|------|
| **API** | test_api_endpoints.py | ヘルスチェック、メトリクス、OpenAPI、404、エラー形式 |
| **セキュリティ** | test_security.py | セキュリティヘッダー、CORS |
| **契約** | contract/test_contract_auth.py | ログインAPI、/me、認証エラー、ヘルスチェック |
| **契約** | contract/test_contract_mlops.py | MLOps パイプラインAPI |
| **契約** | contract/test_contract_chaos.py | Chaos エンドポイント |
| **統合** | test_integration.py | MLOps、認証フロー |
| **DB** | test_database.py | データベース接続 |
| **レート制限** | test_rate_limit.py | レート制限 |
| **キャッシュ** | test_cache.py | キャッシュ |

### E2E（e2e/）

| テスト | 内容 |
|--------|------|
| login.spec.ts | ログイン画面の E2E |
| chaos.spec.ts | Chaos Engineering ページの E2E |

### CI/CD

- `.github/workflows/ci-cd.yml` に Lint・テスト・デプロイのパイプラインあり
- バックエンドテストは `if: false` で一時スキップ（依存関係のため）

## 企業が信用するためのテスト観点

| 観点 | 現状 | 推奨 |
|------|------|------|
| **認証・認可** | 契約テストあり | ログイン・トークン・RBAC の自動テスト |
| **API 契約** | 契約テストあり | 主要APIのリクエスト/レスポンス形式の検証 |
| **セキュリティ** | ヘッダー・CORS テストあり | OWASP チェック、脆弱性スキャン |
| **負荷** | load_test_locust.py あり | スループット・レイテンシの基準値 |
| **E2E** | Playwright あり | 主要画面の動作確認 |
| **カバレッジ** | pytest-cov 設定済み | 70% 以上を目標 |

---

## テスト実行方法

```bash
# バックエンド（backend ディレクトリで）
cd backend
python -m pytest tests/ -v --no-cov

# 契約テストのみ
python -m pytest tests/contract/ -v --no-cov

# フロントエンド
cd frontend
npm test

# E2E（Playwright）
cd e2e
npx playwright test
```

---

## 面談での説明ポイント

- **契約テスト**: API の入出力形式を自動検証し、フロント・バック間の整合性を保証
- **セキュリティテスト**: ヘッダー・CORS の設定を検証
- **CI/CD**: プッシュ時に Lint・テストを自動実行するパイプラインを構築済み
