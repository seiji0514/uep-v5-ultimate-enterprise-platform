# 100システム一覧 抽出作業 - ノートPC用 指示書

**目的**: ノートPC でデスクトップPC と同様の抽出作業を行い、100システム一覧を更新・確認する。

---

## 前提条件

- ノートPC に `uep-v5-ultimate-enterprise-platform` プロジェクトが存在すること
- Cursor または VSCode が利用可能であること
- PowerShell または コマンドプロンプトが利用可能であること

---

## 作業手順

### 1. プロジェクトを開く

1. Cursor（または VSCode）を起動
2. メニュー「ファイル」→「フォルダーを開く」
3. ノートPC 上のプロジェクトパスを指定  
   （例: `C:\Users\<ユーザー名>\...\uep-v5-ultimate-enterprise-platform`）

---

### 2. AIエージェントに依頼する

Cursor の AI チャットで、次のメッセージを送信する：

```
以下の作業をお願いします。

1. このPC（ノートPC）のプロジェクト内から、100システム一覧を抽出してください。
2. docs/100システム一覧_参照用.md を、ノートPC 分の抽出結果で更新してください。
3. デスクトップPC とノートPC の違いがある場合は、ノートPC 分として追記するか、両方の情報を統合してください。

抽出対象：
- backend ディレクトリ内のモジュール（ディレクトリ名）
- docker-compose で定義されているサービス名
- medical, space, traffic, projects, infrastructure, tools 配下のプロジェクト・ディレクトリ
- その他、独立したシステム・モジュール・API として識別できるもの

参考：docs/100システム一覧_参照用.md の既存フォーマットに従ってください。
```

---

### 3. 手動で抽出する場合（AI が使えない場合）

PowerShell で以下を実行し、結果をメモする。

```powershell
# プロジェクトルートに移動
cd C:\Users\<ユーザー名>\<プロジェクトパス>\uep-v5-ultimate-enterprise-platform

# バックエンドモジュール一覧
Get-ChildItem -Path ".\backend" -Directory | Where-Object { $_.Name -notmatch "pycache|pytest|venv" } | Select-Object -ExpandProperty Name

# プロジェクトディレクトリ一覧（medical, space, traffic, projects）
Get-ChildItem -Path ".\medical" -Directory -ErrorAction SilentlyContinue | Select-Object -ExpandProperty Name
Get-ChildItem -Path ".\space" -Directory -ErrorAction SilentlyContinue | Select-Object -ExpandProperty Name
Get-ChildItem -Path ".\traffic" -Directory -ErrorAction SilentlyContinue | Select-Object -ExpandProperty Name
Get-ChildItem -Path ".\projects" -Directory -ErrorAction SilentlyContinue | Select-Object -ExpandProperty Name
```

---

### 4. 抽出結果の反映

1. `docs/100システム一覧_参照用.md` を開く
2. ノートPC で見つかった項目・差分がある項目を追記または修正
3. ドキュメント冒頭に「ノートPC 抽出日: YYYY-MM-DD」を追記（任意）

---

### 5. デスクトップPC との統合（任意）

- デスクトップPC とノートPC でプロジェクト構成が異なる場合
- 両方の結果を統合する場合は、セクションを分けて記載する  
  （例: 【デスクトップPC抽出】、【ノートPC抽出】）

---

## 確認事項

- [ ] プロジェクトがノートPC に存在する
- [ ] 抽出結果が `docs/100システム一覧_参照用.md` に反映された
- [ ] デスクトップPC との差分がある場合は追記・注記した

---

## 補足

- ノートPC にプロジェクトが一部しかない場合（例: サブセットのみクローン）は、その旨をドキュメントに注記する
- 抽出は「デスクトップ分で」という前提がある場合、ノートPC はデスクトップのコピー・同期先として扱い、差分確認のみ行う運用も可能

---

## オプション: Chrome ブックマークから抽出

プロジェクトに存在しないが、ブックマークに残っているシステム名を抽出する場合：

### 方法A: Chrome を終了しない（推奨）

1. Chrome で **Ctrl + Shift + O** でブックマークマネージャーを開く
2. 右上 **⋮** → **ブックマークをエクスポート**
3. HTML ファイルを保存（例: プロジェクトの `bookmarks.html`）
4. プロジェクトルートで実行:
   ```
   python scripts/extract_chrome_bookmarks.py --html bookmarks.html
   ```
5. 出力: `docs/chrome_bookmarks_抽出結果.md` にタイトル・URL 一覧が保存される

### 方法B: Chrome を終了する

1. Chrome を終了
2. `python scripts/extract_chrome_bookmarks.py` を実行
