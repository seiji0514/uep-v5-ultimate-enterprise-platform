## 2. 追加スキル・技術案（優先度順）

### 高優先度（実務ニーズが高い）

| スキル・技術 | 追加内容 | 参画メリット |
|--------------|----------|--------------|
| **Rust** | 高性能ミドルウェア、WASM モジュール | メモリ安全、パフォーマンス、WebAssembly |
| **gRPC** | 既存 proto の拡張、ストリーミング RPC | マイクロサービス間通信、型安全 API |
| **WebAssembly (WASM)** | ブラウザ内推論、エッジ推論 | オンデバイス AI、低レイテンシ |
| **Terraform** | 既存 `cloud_infra/iac.py` の拡張、モジュール化 | IaC 実務、マルチクラウド |
| **Argo CD / Flux** | GitOps パイプライン | 宣言的デプロイ、Kubernetes 運用 |
| **FinOps** | クラウドコスト可視化 API、予測 | コスト最適化、実務ニーズ |

### 中優先度（差別化・学習価値）

| スキル・技術 | 追加内容 | 参画メリット |
|--------------|----------|--------------|
| **Apache Flink** | ストリーム処理、Kafka 連携 | リアルタイム処理、イベント駆動の深化 |
| **Apache Airflow** | DAG オーケストレーション、ML パイプライン | データパイプライン、スケジューリング |
| **Redis Streams** | 軽量イベントストリーム、Kafka 代替 | シンプルなストリーミング、キャッシュ連携 |
| **Ollama / llama.cpp** | ローカル LLM 推論、オンデバイス | プライバシー、オフライン推論 |
| **LangGraph** | エージェント、マルチステップ推論 | 生成 AI の高度化、ワークフロー |
| **WebSocket 双方向** | リアルタイム通知、チャット | 既存 WebSocket の拡張、リアルタイム UI |
| **tRPC** | 型安全 API（TypeScript 連携） | フロント・バックエンド型共有 |
| **Pydantic v2** | 既存モデルの移行、パフォーマンス | バリデーション高速化 |

### 低優先度（将来・興味ベース）

| スキル・技術 | 追加内容 | 参画メリット |
|--------------|----------|--------------|
| **Deno** | スクリプト実行、エッジ関数 | モダン TypeScript ランタイム |
| **Elixir / Phoenix** | リアルタイム機能、WebSocket | 並行処理、フォールトトレランス |
| **Solid.js / Svelte** | 軽量フロントエンド | バンドルサイズ削減、パフォーマンス |
| **Drizzle ORM** | TypeScript 用 ORM | 型安全 DB アクセス |
| **Temporal** | ワークフローオーケストレーション | 長時間実行、リトライ、Saga の代替 |
| **OpenTelemetry Collector** | トレース・メトリクス集約 | 可観測性の統一 |
| **Backstage** | 開発者ポータル、カタログ | プラットフォームエンジニアリング |

## 6. 参画者が取り組む際の進め方

1. **興味・実務に合わせて選択** … 上記から1〜2つを選ぶ
2. **既存モジュールとの連携** … 単体ではなく UEP の他コンポーネントと連携させる
3. **ドキュメント整備** … ENHANCEMENTS.md 形式で実装内容を記録
4. **テスト・CI 追加** … 新機能にはテストと CI パイプラインへの組み込みを推奨
