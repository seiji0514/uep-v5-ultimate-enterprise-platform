# 公開用リポジトリ 準備ガイド

**目的**: UEP v5.0 を GitHub 等で公開する際の準備手順と注意事項。

---

## 1. 事前確認

### 1.1 公開範囲の決定

- [ ] 公開するモジュール・ディレクトリを特定する
- [ ] 非公開にすべきファイル（認証情報、個人情報、社外秘）を洗い出す
- [ ] `.gitignore` の見直し

### 1.2 除外対象の例

| 対象                     | 理由                     |
| ------------------------ | ------------------------ |
| `.env`                   | 環境変数・シークレット   |
| `data/production_users.json` | 本番ユーザー情報     |
| `*.pem`, `*.key`         | 秘密鍵                   |
| 企業固有の設定・ドキュメント | 機密情報             |

---

## 2. 準備手順

### 2.1 機密情報の除去

```powershell
# .env のサンプル化（実際の値は空にする）
Copy-Item .env .env.example
# .env.example の値をプレースホルダに置換
```

### 2.2 .gitignore の確認

以下が含まれていることを確認：

```
.env
*.pem
*.key
data/production_users.json
__pycache__/
node_modules/
.venv/
```

### 2.3 README の整備

- [ ] プロジェクト概要
- [ ] 起動手順（ローカル開発用）
- [ ] 必要な環境（Python, Node.js, Docker 等）
- [ ] ライセンス表記

---

## 3. ドキュメントの整理

### 3.1 公開用ドキュメント

| ドキュメント                   | 用途                         |
| ------------------------------ | ---------------------------- |
| docs/100システム一覧_公開用.md | 職務経歴・ポートフォリオ用   |
| docs/起動方法と使用方法.md     | 利用者向けガイド             |

### 3.2 非公開ドキュメント

面談準備、企業固有のメモ等は公開リポジトリに含めない。

---

## 4. リポジトリ作成・プッシュ

### 4.1 新規リポジトリ作成

1. GitHub で新規リポジトリを作成
2. 「Private」または「Public」を選択
3. README は既存のため「Add a README」は不要

### 4.2 プッシュ

```powershell
git remote add origin <リポジトリURL>
git branch -M main
git push -u origin main
```
