1. AI用語集（面談で聞かれやすい用語）

AI用語の意味を聞かれたとき、簡潔に説明してから実装経験を補足。

RAG（ラグ、Retrieval-Augmented Generation）

意味: 検索拡張生成。外部知識を検索してLLMの入力に加え、根拠のある回答を生成。ハルシネーション軽減。

使い方: 「RAGパターンでLangChainとChromaDBを組み合わせ、社内ドキュメント検索を実装しました。」

CoT（コット、Chain-of-Thought）推論

意味: 思考の連鎖。段階的に推論してから結論を出す手法。複雑な問題で精度向上。

使い方: 「CoT推論をプロンプトに組み込み、論理的な回答を導くように設計しました。」

MLOps / LLMOps

意味: MLモデル／LLMの運用管理。実験管理、モデルレジストリ、デプロイ、監視。

使い方: 「MLflowで実験管理、Prometheus/Grafanaで監視基盤を構築しました。」

インクルーシブ雇用AI

意味: 障害者雇用マッチング、アクセシビリティ特化AI、当事者視点UX評価を統合したプラットフォーム。

使い方: 「身体障害者としての当事者視点を活かし、障害者雇用・アクセシビリティに特化したシステムを実装しました。」

アクセシビリティ

意味: 誰もが使いやすい設計。WCAG準拠、スクリーンリーダー対応等。

使い方: 「Webアクセシビリティ検査員検定L1～3（書面あり）を取得し、誰もが使いやすいAIシステムの開発に貢献できます。」

Git のブランチ戦略

意味: 複数人で同じリポジトリを扱う際のブランチの分け方・運用ルール。main（本番）、develop（開発）、feature/機能名（新機能）など。PR（プルリクエスト）でレビューしてからマージ。

使い方: 「feature ブランチでの開発、PR によるマージの流れは理解しています。実務での経験は浅いので、入社後は実践しながら学びます。」

チーム開発のフロー

意味: 企画→設計→タスク分割→実装→コードレビュー→マージ・テスト→デプロイ→運用。役割分担と連携の流れ。

使い方: 「チーム実務経験は浅いですが、一方で、個人プロジェクトでも企画から運用まで一般的な流れを網羅しています。フローの理解と実装力はあり、貢献できると考えています。」

7. デモ手順

進め方 実績を聞かれたら「画面共有でデモできます」と伝え、了承されたら実施。

起動 ローカル: start-backend.bat、cd frontend → npm start。フロントエンド: http://localhost:3000。バックエンド: http://localhost:8000 または http://localhost:8080（8000がWindows予約の場合は8080に自動切り替え）。API Docs: http://localhost:8000/docs または http://localhost:8080/docs。クラウド: https://uep-v5-ultimate-enterprise-platform.vercel.app（事前に /health にアクセスしてRenderを起こす）

強調ポイント 実際に動作するシステム／稼働率99.995%・推論レイテンシ<10ms／MLOps・生成AI・インクルーシブ雇用AI／Level 1～5 統合／Chaos Engineering・GraphQL・Contract Testing

デモンストレーションの流れ（時間に応じて3段階）

【5分程度】最小限デモ

  1. システム概要（1-2分） 口頭でUEP（ユーイーピー）v5.0の概要（Level 1～5、稼働率99.995%、100システム以上）
  2. ログイン・ダッシュボード（2-3分） http://localhost:3000 でログイン → ダッシュボード一覧を表示し、Level 1～5 の構成を説明

【10分程度】標準デモ

上記5分に加えて:

  1. バックエンドAPI（2-3分） Swagger UI（スワッガーユーアイ）で主要API（エーピーアイ）を一覧表示、auth・mlops・inclusive-work 等を説明
  2. インクルーシブ雇用AI（2-3分） マッチング・アクセシビリティAI・UX評価の画面を簡潔に実演

【15-20分程度】フルデモ

上記10分に加えて:

  1. Chaos Engineering（カオスエンジニアリング） 遅延注入・エラー注入の実行
  2. GraphQL（グラフキューエル） hello / health / services クエリ実行
  3. WebAssembly（ウェブアセンブリ） 加算・乗算・合計の計算デモ
  4. 技術スタック・アーキテクチャ・品質指標 口頭で補足
