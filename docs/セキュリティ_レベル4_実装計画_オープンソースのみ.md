# セキュリティ・サイバー対策 レベル4 実装計画

**作成日**: 2026年3月2日  
**対象**: UEP v5.0 統合セキュリティ基盤  
**方針**: オープンソース・無料ツールのみ（有料ツール不使用）

---

## 1. 概要

### 1.1 目的

UEP v5.0 のセキュリティコマンドセンターを、エンタープライズ規模のサイバー対策基盤へ拡張する。IDS/IPS、EDR、SIEM、脅威インテリジェンス、コンプライアンスレポートを、**オープンソース・無料ツールのみ**で実現する。

### 1.2 採用ツール一覧（すべて無料）

| 領域 | ツール | ライセンス | 役割 |
|------|--------|------------|------|
| **IDS/IPS** | Suricata | GPL | ネットワーク侵入検知・防止 |
| **ログ解析** | Zeek | BSD | ネットワークトラフィック解析 |
| **EDR** | Wazuh | GPL v2 | エンドポイント検知・対応 |
| **SIEM/ログ** | OpenSearch + OpenSearch Dashboards | Apache 2.0 | ログ集約・検索・可視化 |
| **代替SIEM** | Grafana Loki + Grafana | AGPL | 軽量ログ基盤 |
| **脅威インテリジェンス** | MISP | AGPL | 脅威情報の共有・管理 |
| **可視化** | Grafana | AGPL | ダッシュボード・アラート |
| **メッセージキュー** | Apache Kafka | Apache 2.0 | イベントストリーミング（既存） |

## 2. Phase 別実装計画

### Phase 1: IDS/IPS 連携（1〜2ヶ月）

#### 2.1.1 目標

- Suricata によるネットワーク侵入検知
- アラートを UEP Security Center に取り込み・表示
- 既存 SecurityEvent モデルへの `source: "suricata"` 追加

#### 2.1.2 実装内容

| 項目 | 内容 |
|------|------|
| **Suricata 導入** | Docker コンテナ、ルールセット（Emerging Threats 無料版） |
| **アラート取り込み** | Suricata eve.json を Filebeat/Fluentd または Python で監視 |
| **API 拡張** | `POST /api/v1/cyber-defense/suricata/alerts` でイベント登録 |
| **Security Center 連携** | `event_type: "ids_alert"`, `metadata.suricata_rule_id` 等 |

#### 2.1.3 ディレクトリ構成

```
backend/cyber_defense/
├── __init__.py
├── routes.py              # /api/v1/cyber-defense/*
├── suricata_client.py      # Suricata アラート取得・正規化
├── models.py              # SuricataAlert, WazuhAlert 等
└── services/
    └── suricata_ingest.py  # eve.json パース・SecurityMonitor 連携

infrastructure/cyber-defense/
├── suricata/
│   ├── suricata.yaml
│   └── rules/             # ルール配置
└── docker-compose.cyber-defense.yml
```

#### 2.1.4 docker-compose 追加サービス

```yaml
# docker-compose.cyber-defense.yml
suricata:
  image: jasonish/suricata:latest
  volumes:
    - ./infrastructure/cyber-defense/suricata/suricata.yaml:/etc/suricata/suricata.yaml
    - ./infrastructure/cyber-defense/suricata/rules:/var/lib/suricata/rules
    - suricata-logs:/var/log/suricata
  network_mode: host  # またはパケットキャプチャ用設定
  cap_add:
    - NET_RAW
    - NET_ADMIN
```

### Phase 2: SIEM 連携（2〜3ヶ月）

#### 2.2.1 目標

- OpenSearch によるログ集約・検索
- アプリケーションログ、Suricata ログ、認証ログの一元管理
- Grafana または OpenSearch Dashboards で可視化

#### 2.2.2 実装内容

| 項目 | 内容 |
|------|------|
| **OpenSearch 導入** | Docker コンテナ、単一ノード構成 |
| **ログ収集** | OpenSearch 用 Filebeat / Logstash または Fluent Bit |
| **UEP 連携** | 監査ログを Kafka → OpenSearch に送信 |
| **API** | `GET /api/v1/cyber-defense/siem/search?query=...` 検索プロキシ |

#### 2.2.3 代替軽量構成

**Grafana Loki** を採用する場合（リソース節約）:

- Loki: ログストレージ
- Promtail: ログ収集
- Grafana: 可視化（既存 Prometheus と統合可能）

---

### Phase 3: EDR 連携（2〜3ヶ月）

#### 2.3.1 目標

- Wazuh によるエンドポイント監視
- ファイル整合性監視（FIM）、プロセス監視、マルウェア検知
- アラートを UEP Security Center に取り込み

#### 2.3.2 実装内容

| 項目 | 内容 |
|------|------|
| **Wazuh 導入** | Wazuh Manager（Docker）、Wazuh Agent（監視対象） |
| **アラート取り込み** | Wazuh API または Webhook で UEP に通知 |
| **API** | `POST /api/v1/cyber-defense/wazuh/alerts` |
| **Security Center 連携** | `event_type: "edr_alert"`, `metadata.wazuh_rule_id` |

#### 2.3.3 構成

```
Wazuh Manager (Docker)
    ├── Wazuh Agent (サーバー/コンテナ)
    └── UEP Backend (Webhook 受信)
```

---

### Phase 4: 脅威インテリジェンス連携（1〜2ヶ月）

#### 2.4.1 目標

- MISP による脅威情報の管理
- IP アドレス、ドメイン、ハッシュの IOC 照合
- Suricata ルールの自動更新（オプション）

#### 2.4.2 実装内容

| 項目 | 内容 |
|------|------|
| **MISP 導入** | Docker コンテナ、初期設定 |
| **IOC 照合 API** | `POST /api/v1/cyber-defense/threat-intel/check` |
| **IP/ドメイン/ハッシュ** | MISP API で照合、結果を返却 |
| **Security Center 連携** | 照合結果がヒットした場合 SecurityEvent 登録 |

---

### Phase 5: コンプライアンスレポート（1〜2ヶ月）

#### 2.5.1 目標

- 監査ログの出力
- SOC2、ISO27001、NIST CSF 等のレポート雛形

#### 2.5.2 実装内容

| 項目 | 内容 |
|------|------|
| **監査ログ** | 操作ログ、アクセスログの構造化 |
| **レポート API** | `GET /api/v1/cyber-defense/compliance/report?format=json` |
| **レポート種別** | アクセスログサマリ、インシデント一覧、セキュリティイベント統計 |

## 3. 既存 UEP との統合

### 3.1 Security Center 拡張

| 既存 | 拡張 |

|------|------|
| `SecurityEvent` | `source: "suricata" | "wazuh" | "internal"` 追加 |
| `SecurityMonitor.log_event()` | 外部連携イベントの登録 |
| `SecurityCenterPage.tsx` | 新規タブ「サイバー対策」追加 |

### 3.2 新規 API エンドポイント

```
GET  /api/v1/cyber-defense/overview           # ダッシュボード概要
GET  /api/v1/cyber-defense/suricata/alerts    # Suricata アラート一覧
POST /api/v1/cyber-defense/suricata/alerts    # Suricata アラート取り込み
GET  /api/v1/cyber-defense/wazuh/alerts      # Wazuh アラート一覧
POST /api/v1/cyber-defense/wazuh/alerts      # Wazuh Webhook 受信
POST /api/v1/cyber-defense/threat-intel/check # IOC 照合
GET  /api/v1/cyber-defense/compliance/report # コンプライアンスレポート
```

### 3.3 フロントエンド拡張

| コンポーネント | 内容 |
|----------------|------|
| `CyberDefensePage.tsx` | 新規ページ（サイバー対策ダッシュボード） |

## 6. 前提条件・注意事項

### 6.1 前提

- Docker / Docker Compose 環境
- 本番相当のネットワーク（パケットキャプチャ用の権限）
- 運用体制（アラート対応、ログ保持期間の検討）

### 6.2 注意事項

- **Suricata**: `network_mode: host` またはパケットミラーリングが必要な場合あり
- **Wazuh Agent**: 監視対象サーバーへのインストールが必要
- **OpenSearch**: 本番ではクラスター構成を推奨（本計画は単一ノード）
- **MISP**: 初期設定（管理者アカウント、フィード登録）が必要
