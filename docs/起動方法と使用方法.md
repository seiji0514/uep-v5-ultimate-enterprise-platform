## システム構成

| システム | ポート | 役割 |
|----------|--------|------|
| **UEP v5.0 フロントエンド** | 3000 | メインUI（ダッシュボード、各機能画面） |
| **UEP v5.0 バックエンド** | 8080 | API（MLOps、生成AI、監視、クラウドインフラ等） |
| **統合セキュリティ・防衛プラットフォーム フロントエンド** | 3001 | セキュリティ専用UI |
| **統合セキュリティ・防衛プラットフォーム バックエンド** | 9001 | セキュリティAPI（IDS/IPS, EDR, SIEM等） |
| **企業横断オペレーション基盤 フロントエンド** | 3020 | 観測・タスク・リスク管理UI |
| **企業横断オペレーション基盤 バックエンド** | 9020 | EOH API |

※ ポート8000はWindows予約範囲（7938-8037）に含まれるため、UEPバックエンドは8080を使用します。

## UEP v5.0 起動方法

### 前提条件

- Python 3.11以上
- Node.js 18以上（フロントエンド用）
- npm

### 1. バックエンドの起動

**コマンドプロンプト:**
```cmd
cd C:\uep-v5-ultimate-enterprise-platform
.\start-backend.bat
```

**PowerShell:**
```powershell
cd C:\uep-v5-ultimate-enterprise-platform\backend
python main.py
```

- **URL**: http://localhost:8080
- **API Docs**: http://localhost:8080/docs
- **停止**: Ctrl+C

### 2. フロントエンドの起動

**PowerShell（推奨）:**
```powershell
cd C:\uep-v5-ultimate-enterprise-platform
.\start-frontend.ps1
```

**コマンドプロンプト:**
```cmd
cd C:\uep-v5-ultimate-enterprise-platform
.\start-frontend.bat
```

**直接 npm で起動:**
```powershell
cd C:\uep-v5-ultimate-enterprise-platform\frontend
npm install   # 初回のみ
npm start
```

- **URL**: http://localhost:3000
- **停止**: Ctrl+C

### 起動順序

1. バックエンド（8080）を先に起動
2. フロントエンド（3000）を起動

## 統合セキュリティ・防衛プラットフォーム 起動方法

UEP v5.0とは**別の個別システム**です。UEP起動時には含まれません。

### 前提条件

- UEP の `backend/venv` が作成済みであること（初回は UEP の `start-backend.bat` を1回実行して venv を作成）
- Node.js 18以上（フロントエンド用）

### 1. バックエンドの起動

```cmd
cd C:\uep-v5-ultimate-enterprise-platform\security_defense_platform
.\start-backend.bat
```

- **URL**: http://localhost:9001
- **API Docs**: http://localhost:9001/docs
- **停止**: Ctrl+C

※ ポート9001が使用中の場合は、9002〜9010の空きポートで自動起動します。

### 2. フロントエンドの起動

**PowerShell:**
```powershell
cd C:\uep-v5-ultimate-enterprise-platform\security_defense_platform
.\start-frontend.ps1
```

**コマンドプロンプト:**
```cmd
cd C:\uep-v5-ultimate-enterprise-platform\security_defense_platform
.\start-frontend.bat
```

**直接 npm で起動:**
```powershell
cd C:\uep-v5-ultimate-enterprise-platform\security_defense_platform\frontend
npm install   # 初回のみ
npm start
```

- **URL**: http://localhost:3001
- **停止**: Ctrl+C

### 起動順序

1. バックエンド（9001）を先に起動
2. フロントエンド（3001）を起動

## 企業横断オペレーション基盤 起動方法

観測・タスク・リスクを一元管理する独立システム。UEPダッシュボードからリンク可能。

- **DB**: スキーマ変更時は `enterprise_operations_hub/eoh.db` を削除して再起動
- **UEP連携**: UEPの `.env` に `REACT_APP_EOH_URL=http://localhost:3020` を設定

### 1. バックエンドの起動

```powershell
cd C:\uep-v5-ultimate-enterprise-platform\enterprise_operations_hub
.\start-backend.bat
```

- **URL**: http://localhost:9020
- **API Docs**: http://localhost:9020/docs

### 2. フロントエンドの起動

```powershell
cd C:\uep-v5-ultimate-enterprise-platform\enterprise_operations_hub\frontend
npm install   # 初回のみ
npm start
```

- **URL**: http://localhost:3020
- **ログイン**: kaho0525 / 0525

## UEP v5.0 使用方法

### ログイン

1. ブラウザで http://localhost:3000 を開く
2. ログインページで以下いずれかを入力

| ユーザー名 | パスワード | 用途 |
|------------|------------|------|
| kaho0525 | 0525 | 管理者 |
| developer | dev123 | 開発者 |
| viewer | view123 | 閲覧者 |

3. 「ログイン」または「デモ: KAH00525 / KAH0052514 または開発者 / DEV123」ボタンをクリック

### 主な機能

- **ダッシュボード**: 統合状況の概要
- **プロジェクト一覧**: プロジェクト管理
- **MLOps**: 機械学習モデル管理・実験追跡
- **生成AI**: RAG、チャット、推論
- **クラウドインフラ**: IaC、オーケストレーション
- **インフラ構築**: 専用構築ツール
- **IDOP**: 統合開発・運用
- **AI支援開発**: コード補助、ドキュメント生成
- **ERP**: 統合基幹業務
- **製造・IoT**: OPC-UA連携
- **FinTech**: 金融・ストレステスト
- **エネルギー**: エネルギー管理
- **医療**: FHIR連携
- **その他**: GraphQL、Chaos Engineering、個人会計 等

### API ドキュメント

- http://localhost:8080/docs （Swagger UI）

## 統合セキュリティ・防衛プラットフォーム 使用方法

### ログイン

1. ブラウザで http://localhost:3001 を開く
2. ログインページで以下いずれかを入力

| ユーザー名 | パスワード |
|------------|------------|
| kaho0525 | kaho052514 |
| developer | dev123 |

3. 「ログイン」をクリック

### タブ機能

| タブ | 機能 | 説明 |
|------|------|------|
| **イベント** | セキュリティイベント一覧 | 脅威レベルでフィルタ可能。送信元・送信先・発生日時を表示 |
| **インシデント** | インシデント一覧 | 重大度・ステータスでフィルタ可能 |
| **リスク** | リスク分析 | カテゴリ、発生確率、影響度、対策を表示 |
| **サイバー概要** | ダッシュボード | Suricata/Wazuh/セキュリティイベントのサマリ、連携ステータス |
| **IDS/IPS** | Suricataアラート | ネットワーク侵入検知・防止のアラート一覧 |
| **EDR** | Wazuhアラート | エンドポイント検知・対応のアラート一覧 |
| **脅威インテリ** | IOC照合 | IP・ドメイン・ハッシュを照合し、既知の脅威か判定。デモ: 192.168.1.100, 10.0.0.99, malware.example.com |
| **コンプライアンス** | レポート | 期間内のイベント数、Critical/High件数、推奨事項 |
| **SIEM検索** | ログ検索 | キーワードでイベントを検索 |

### 操作のヒント

- **更新ボタン**: 各タブの「更新」でデータを再取得
- **フィルタ**: イベント・インシデントタブでドロップダウンから条件を選択
- **脅威インテリ**: 照合ボタンでIOCをチェック
- **SIEM検索**: キーワード入力後、検索ボタンまたはEnterで実行

### API ドキュメント

- http://localhost:9001/docs （Swagger UI）

## トラブルシューティング

### ポートが使用できない（WinError 10013 / 10048）

- **8000**: Windows予約範囲のため使用不可。UEPバックエンドは8080を使用
- **9001使用中**: 統合セキュリティ・防衛プラットフォームは9002〜9010で自動的に空きポートを探して起動

### フロントエンドが起動しない（PowerShellでバッチ実行時）

- `.\start-frontend.ps1` を使用（PowerShell用）
- または `cd frontend` のうえで `npm start` を直接実行

### 認証エラー（Not authenticated）

- ログイン画面でデモアカウントでログイン
- バックエンドが起動しているか確認（8080 または 9001）

### データが表示されない

- バックエンドが起動しているか確認
- ブラウザの開発者ツール（F12）でネットワークエラーを確認
- CORSエラーの場合、バックエンドのCORS設定を確認
