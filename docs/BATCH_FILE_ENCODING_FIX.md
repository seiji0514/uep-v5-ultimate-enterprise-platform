# バッチファイルの文字化け修正

**作成日**: 2026年1月29日

---

## 🔧 問題

Windowsのバッチファイル（`.bat`）で日本語が文字化けする問題が発生していました。

---

## ✅ 解決方法

各バッチファイルの先頭に `chcp 65001 >nul` を追加しました。

これにより、UTF-8エンコーディングで正しく日本語が表示されます。

### 修正されたファイル

- ✅ `start-all.bat`
- ✅ `start-backend.bat`
- ✅ `start-frontend.bat`
- ✅ `stop-all.bat`

---

## 📝 説明

### `chcp 65001` とは

- `chcp`: コードページ（文字エンコーディング）を変更するコマンド
- `65001`: UTF-8のコードページ番号
- `>nul`: 出力を非表示にする（エラーメッセージを抑制）

### なぜ必要か

Windowsコマンドプロンプトは、デフォルトでShift-JIS（CP932）を使用しますが、ファイルがUTF-8で保存されている場合、日本語が文字化けします。

`chcp 65001`を実行することで、UTF-8に切り替え、正しく日本語が表示されます。

---

## 🚀 使用方法

修正後、通常通り使用できます：

```cmd
start-all.bat
```

日本語が正しく表示されるはずです。

---

## ⚠️ 注意事項

- `chcp 65001`は、そのコマンドプロンプトセッション内でのみ有効です
- 新しいコマンドプロンプトを開いた場合は、再度実行されます（バッチファイル内で自動実行されます）
- 一部の古いWindowsシステムでは、UTF-8が完全にサポートされていない場合があります
