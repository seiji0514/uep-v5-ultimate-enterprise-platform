# Git コミット方法

## 基本手順

### 1. 変更状況の確認
```bash
git status
```

### 2. 変更をステージング
```bash
# 全ファイルを追加
git add -A

# または特定ファイルのみ
git add frontend/src/components/xxx.tsx
```

### 3. コミット
```bash
git commit -m "コミットメッセージ"
```

### 4. リモートへ送信（任意）
```bash
git push origin main
```

---

## よく使うコマンド一覧

| 操作 | コマンド |
|------|----------|
| 状態確認 | `git status` |
| 全変更を追加 | `git add -A` |
| コミット | `git commit -m "メッセージ"` |
| 追加＋コミットを一括 | `git add -A && git commit -m "メッセージ"` |
| プッシュ | `git push` |

---

## コミットメッセージの例

- `面談用: 〇〇機能を追加`
- `fix: 〇〇のエラーを修正`
- `docs: READMEを更新`

---

## 頻度の目安

- **毎日**: 作業履歴が細かく残る
- **機能単位**: 1機能完了ごとにまとめてコミット
- **目安**: 「ここまでで一度保存したい」と思ったタイミングでコミット
