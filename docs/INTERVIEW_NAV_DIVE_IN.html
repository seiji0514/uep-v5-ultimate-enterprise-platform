7. デモ手順（インフラ構成を強調）

進め方 実績を聞かれたら「画面共有でデモできます。インフラ構成もご確認いただけます」と伝え、了承されたら実施。

起動 ローカル: start-backend.bat（スタートバックエンドバット）、cd frontend（シーディーフロントエンド）→ npm start（エヌピーエムスタート）。クラウド: https://uep-v5-ultimate-enterprise-platform.vercel.app（Vercel（バーセル））（事前に /health にアクセスしてRender（レンダー）を起こす）

インフラエンジニア向け強調ポイント

  • API Gateway（エーピーアイゲートウェイ） Kong（コン）、Envoy（エンボイ）の設計・構築
  • 監視基盤 Prometheus（プロメテウス）、Grafana（グラファナ）で稼働率99.995%測定
  • イベントストリーミング Apache Kafka（アパッチカフカ）
  • オブジェクトストレージ MinIO（ミニオー）
  • コンテナ環境 Docker（ドッカー）、Kubernetes（クーバネティス）、Terraform（テラフォーム）
  • セキュリティ Zero Trust（ゼロトラスト）、mTLS（エムティーエルエス）、Vault（ボルト）

デモの流れ（インフラエンジニア向け）

【5分程度】最小限デモ

  1. システム概要（1-2分） 口頭でUEP（ユーイーピー）v5.0の概要。インフラ構成（API Gateway（エーピーアイゲートウェイ）、Kafka（カフカ）、監視基盤、コンテナ環境）を簡潔に
  2. ダッシュボード（2-3分） http://localhost:3000 でログイン → クラウドインフラ、インフラ構築専用、MLOps（エムエルオープス）、監視の画面を説明

【10分程度】標準デモ

上記5分に加えて:

  1. Swagger UI（スワッガーユーアイ）（2-3分） http://localhost:8000/docs でAPI（エーピーアイ）一覧。auth（オース）、mlops（エムエルオープス）、cloud-infra（クラウドインフラ）、Chaos Engineering（カオスエンジニアリング）等を説明
  2. インフラ構築専用（2-3分） サイドメニュー「インフラ構築」→ 設計→構築→デプロイ→検証のワークフロー、ブループリント、パイプラインを実演

【15-20分程度】フルデモ

上記10分に加えて:

  1. Chaos Engineering（カオスエンジニアリング） 遅延注入・エラー注入でレジリエンス検証
  2. GraphQL（グラフキューエル） hello / health / services クエリ実行
  3. 技術スタック・アーキテクチャ マイクロサービス、イベント駆動、API Gateway（エーピーアイゲートウェイ）（Kong（コン）、Envoy（エンボイ））、監視（Prometheus（プロメテウス）、Grafana（グラファナ））、Kafka（カフカ）、MinIO（ミニオー）、Docker（ドッカー）、Kubernetes（クーバネティス）、Terraform（テラフォーム）を口頭で

インフラ構成の口頭説明（デモ前の概要用）

「UEP（ユーイーピー）v5.0のインフラ構成をご説明します。API Gateway（エーピーアイゲートウェイ）はKong（コン）とEnvoy（エンボイ）で設計・構築。イベントストリーミングはApache Kafka（アパッチカフカ）。オブジェクトストレージはMinIO（ミニオー）。監視はPrometheus（プロメテウス）とGrafana（グラファナ）で稼働率99.995%を測定。コンテナはDocker（ドッカー）とKubernetes（クーバネティス）、IaC（アイアーシー）はTerraform（テラフォーム）。セキュリティはZero Trust（ゼロトラスト）とmTLS（エムティーエルエス）、Vault（ボルト）です。画面共有で実際の構成をご確認いただけます。」

デモの最終案（口頭台本・画面を見せながら）

①→②→③→⑤→⑥→⑦→④の順で実施。時間に応じて⑤〜⑦は省略可。

② クラウドインフラの説明

「こちらがクラウドインフラの画面です。クラウド設計・運用、IaC（アイアーシー）運用、コンテナ化・オーケストレーションを担当するモジュールです。タブは3つあります。リソースでデプロイ済みのクラウドリソースを一覧表示、IaC（アイアーシー）テンプレートでTerraform（テラフォーム）などのテンプレートを管理、デプロイメントでデプロイ履歴を確認できます。」

③ インフラ構築専用システムの説明

「こちらがインフラ構築専用システムです。設計→構築→デプロイ→検証のワークフローでインフラ構築を管理し、Docker（ドッカー）、Kubernetes（クーバネティス）、Terraform（テラフォーム）に対応しています。構築プロジェクトが2件あります。1つ目はWeb（ウェブ）アプリ基盤の構築で、Nginx（エンジンエックス）とRedis（レディス）のコンテナ基盤をDocker（ドッカー）で構築中です。2つ目はKubernetes（クーバネティス）クラスタの構築で、K8s（ケーエイツ）マニフェストによるデプロイを設計段階で管理しています。ブループリントとパイプライン実行で、設計テンプレートと実行履歴を管理しています。」

⑤ Chaos Engineering（カオスエンジニアリング）

「Chaos Engineering（カオスエンジニアリング）では、遅延注入とエラー注入でレジリエンスを検証しています。障害を意図的に発生させ、システムがどのように復旧するかを確認する仕組みです。Swagger UI（スワッガーユーアイ）の chaos（カオス）エンドポイントから実行できます。」

⑥ GraphQL（グラフキューエル）

「GraphQL（グラフキューエル）では、hello（ハロー）、health（ヘルス）、services（サービス）のクエリを実行できます。必要なデータだけを取得するAPI（エーピーアイ）スタイルで、フロントエンドとバックエンドの連携に使っています。」

⑦ 技術スタック・アーキテクチャ（口頭で一気に）

「アーキテクチャはマイクロサービスとイベント駆動です。API Gateway（エーピーアイゲートウェイ）はKong（コン）とEnvoy（エンボイ）、監視はPrometheus（プロメテウス）とGrafana（グラファナ）、イベントストリーミングはKafka（カフカ）、オブジェクトストレージはMinIO（ミニオー）。インフラはDocker（ドッカー）、Kubernetes（クーバネティス）、Terraform（テラフォーム）で構築しています。」

④ まとめ（全体の締め）

「クラウドインフラとインフラ構築専用で設計からデプロイまでを管理し、Chaos Engineering（カオスエンジニアリング）やGraphQL（グラフキューエル）でレジリエンス検証やAPI（エーピーアイ）連携を実現しています。」
