# デモ ボタンエラー対処

**更新**: 2026年2月25日

---

## 面談中ログアウト防止

`backend/.env` に以下を追加すると、トークン有効期限が 120 分になります。

```
ACCESS_TOKEN_EXPIRE_MINUTES=120
```

追加後、バックエンドを再起動してください。

---

## 修正済み

### 1. インフラ構築「実行」ボタン → 403 エラー

**原因**: `manage_infrastructure` 権限が不足していた。

**対応**: デモユーザー（kaho0525、developer）に `manage_infrastructure` を付与済み。

**注意**:
1. **バックエンドを再起動**してください（コード変更の反映）
2. 既にログイン済みの場合は、**一度ログアウトして再ログイン**すると新しい権限が反映されます

## その他の想定エラー

| エラー | 原因 | 対処 |
|--------|------|------|
| バックエンドに接続できません | バックエンド未起動 | `python main.py` で起動 |
| 401 Unauthorized | 未ログイン or トークン期限切れ | ログアウト→再ログイン |
| Chaos「エラー注入」で失敗 | 仕様（確率でエラーを返す） | 正常動作。再試行で成功する場合あり |
| GraphQL 404 | strawberry-graphql 未インストール | `pip install strawberry-graphql[fastapi]` |

---

## 起動後の確認手順

1. バックエンド: `cd backend && python main.py`
2. フロントエンド: `cd frontend && npm start`
3. ログイン: kaho0525 / kaho052514 または developer / dev123
4. インフラ構築 → 実行ボタン → パイプライン実行タブで確認
