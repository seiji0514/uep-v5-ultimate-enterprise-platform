## ⚠️ トラブル時の対応手順

### 問題1: バックエンドが起動しない

**症状**: `start-backend.bat`を実行してもエラーが発生

**対応手順**:

1. Pythonがインストールされているか確認: `python --version`
2. 仮想環境を再作成:
   ```cmd
   cd backend
   rmdir /s /q venv
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```
3. それでも起動しない場合:
   - 「バックエンドの起動に問題が発生しましたが、Swagger UIのスクリーンショットでAPI仕様をご確認いただけます」と説明
   - 事前に準備したスクリーンショットを共有

---

### 問題2: フロントエンドが起動しない

**症状**: `start-frontend.bat`を実行してもエラーが発生

**対応手順**:

1. Node.jsがインストールされているか確認: `node --version`
2. 依存関係を再インストール:
   ```cmd
   cd frontend
   rmdir /s /q node_modules
   npm install
   npm start
   ```
3. それでも起動しない場合:
   - 「フロントエンドの起動に問題が発生しましたが、バックエンドAPIの動作をご確認いただけます」と説明
   - Swagger UIでAPIの動作を実演

---

### 問題3: ログインできない

**症状**: ログイン画面でエラーが発生

**対応手順**:

1. バックエンドが起動しているか確認: `http://localhost:8000/health`
2. ユーザー名とパスワードが正しいか確認:
   - ユーザー名: `kaho0525`
   - パスワード: `kaho052514`
3. ブラウザのコンソールでエラーを確認
4. それでもログインできない場合:
   - 「ログイン機能に問題が発生しましたが、Swagger UIでAPIの動作をご確認いただけます」と説明
   - Swagger UIでAPIの動作を実演

---

### 問題4: 画面共有ができない

**症状**: Google Meetの画面共有ができない

**対応手順**:

1. ブラウザのウィンドウを事前に開いておく
2. 画面共有の代わりに、事前に準備したスクリーンショットを共有
3. または、コードを直接共有して説明
