1. AI用語集（面談で聞かれやすい用語）

前回面談でAI用語の意味を聞かれたため、ラスク面談で使用されそうな用語をピックアップ。意味と使い方を記載。

AX（エーエックス、AI Transformation）

意味: AIによるビジネス変革。企業がAIを活用して業務・プロダクト・組織を変革すること。ラスクのキーワード。

使い方: 「貴社が推進するAXに強く共感しております。AI駆動開発の最前線で貢献したいです。」

RAG（ラグ、Retrieval-Augmented Generation）

意味: 検索拡張生成。外部の知識ベース（ドキュメント・DB（ディービー））を検索し、その結果をLLM（エルエルエム）の入力に加えて回答を生成する手法。ハルシネーション（幻覚）を減らせる。

使い方: 「RAG（ラグ）パターンでLangChain（ラングチェーン）とChromaDB（クローマディービー）を組み合わせ、社内ドキュメント検索システムを実装しました。」

CoT（コット、Chain-of-Thought）推論

意味: 思考の連鎖。LLM（エルエルエム）に「段階的に考えて」と促し、中間の推論ステップを踏ませてから結論を出す手法。複雑な問題で精度が上がる。

使い方: 「CoT（コット）推論をプロンプトに組み込み、論理的な回答を導くように設計しました。」

LLM（エルエルエム、Large Language Model）

意味: 大規模言語モデル。GPT-4（ジーピーティーフォー）、Claude（クロード）、Gemini（ジェミニ）など、大量のテキストで学習したAIモデル。テキスト生成・理解・推論が可能。

使い方: 「LLM（エルエルエム）統合としてOpenAI API（オープンエーアイエーピーアイ）と連携し、自然言語での問い合わせに対応する機能を実装しました。」

MLOps（エムエルオプス、Machine Learning Operations）

意味: ML（エムエル）モデルの開発から本番運用までを一貫して管理する仕組み。実験管理、モデルバージョニング、デプロイ、監視を含む。

使い方: 「MLflow（エムエルフロー）で実験管理とモデルレジストリを構築し、Prometheus（プロメテウス）/Grafana（グラファナ）で監視基盤を整えました。」

LLMOps（エルエルエムオプス）

意味: LLM（エルエルエム）の運用管理。プロンプト管理、バージョニング、A/Bテスト、コスト・レイテンシ監視など。

使い方: 「LLMOps（エルエルエムオプス）としてRAG（ラグ）パイプラインの監視と、推論レイテンシ10ms未満の最適化を実施しました。」

生成AI（セイセイエーアイ、Generative AI）

意味: テキスト・画像・音声などを「生成」するAI。ChatGPT（チャットジーピーティー）、画像生成AIなど。従来の分類・予測型AIと区別される。

使い方: 「生成AIを活用し、RAG（ラグ）とCoT（コット）推論で社内ナレッジ検索システムを構築しました。」

AIエージェント（エーアイエージェント）

意味: 自律的にタスクを実行するAI。ツール呼び出し、複数ステップの推論、外部API（エーピーアイ）連携などを行う。MCP（エムシーピー）/A2A（エーツーエー）はエージェント間連携。

使い方: 「AIエージェントとして、タスクに応じてツールを選択し実行する仕組みを実装しました。」

推論AI（スイロンエーアイ、Inference AI）

意味: 学習済みモデルを使って予測・生成を行うフェーズ。学習（training）と対比される。タスク難度に応じたモデルルーティングも含む。

使い方: 「推論AIとして、簡単な質問は軽量モデル、複雑な質問はGPT-4（ジーピーティーフォー）に振り分けるルーティングを実装しました。」

MCP（エムシーピー、Model Context Protocol）

意味: AIモデルが外部ツール・データソースにアクセスするための標準プロトコル。Anthropic（アンスロピック）等が提唱。

使い方: 「MCP（エムシーピー）により、LLM（エルエルエム）が外部API（エーピーアイ）やデータベースに安全にアクセスできるようにしました。」

A2A（エーツーエー、Agent-to-Agent）

意味: エージェント間連携。複数のAIエージェントが役割分担して協調動作する仕組み。

使い方: 「A2Aで、検索エージェントと要約エージェントを連携させ、効率的な情報処理を実現しました。」

デジタルデバイド（デジタルデバイド）

意味: デジタル技術へのアクセス・活用能力の格差。シニア層などがITに不慣れで取り残される問題。ラスクのシニア向けサービスと直結。

使い方: 「貴社のシニア向けデジタルデバイド解消サービスに、アクセシビリティの専門知識で貢献したいです。」

アクセシビリティ（アクセシビリティ）

意味: 誰もが使いやすい設計。障害者・高齢者・様々な環境でも利用できるUI（ユーアイ）/UX（ユーエックス）。WCAG（ダブリューシーエージー）準拠、スクリーンリーダー対応など。

使い方: 「Webアクセシビリティ検査員検定L1～3を取得し、誰もが使いやすいAIシステムの開発に貢献できます。」

ベクトルデータベース（ベクトルデータベース、Vector DB）

意味: テキストをベクトル（数値配列）に変換して保存し、類似検索ができるDB（ディービー）。RAG（ラグ）で「関連ドキュメントを検索」する際に使用。ChromaDB（クローマディービー）、Pinecone（パインコーン）等。

使い方: 「ChromaDB（クローマディービー）をベクトルデータベースとして使い、RAG（ラグ）で関連ドキュメントを検索する仕組みを実装しました。」

LangChain（ラングチェーン）

意味: LLM（エルエルエム）アプリケーション開発のフレームワーク。RAG（ラグ）、エージェント、チェーンなどを組み立てやすくするPython（パイソン）ライブラリ。

使い方: 「LangChain（ラングチェーン）でRAG（ラグ）チェーンを構築し、ドキュメントの埋め込みと検索を実装しました。」

プロンプトエンジニアリング（プロンプトエンジニアリング）

意味: LLM（エルエルエム）への指示（プロンプト）を設計・最適化すること。Few-shot（フィューショット）、CoT（コット）、役割設定などで出力品質を高める。

使い方: 「プロンプトエンジニアリングで、CoT（コット）推論とFew-shot（フィューショット）例を組み合わせ、回答精度を向上させました。」

チャンキング（チャンキング、Chunking）

意味: 長いテキストを小さな塊（チャンク）に分割すること。RAG（ラグ）ではドキュメントをチャンク化してベクトルDB（ディービー）に格納し、検索対象にする。

使い方: 「チャンキングでドキュメントを適切なサイズに分割し、RAG（ラグ）の検索精度を高めました。」

ハルシネーション（ハルシネーション、Hallucination）

意味: LLM（エルエルエム）が事実と異なる内容を「自信満々に」生成すること。RAG（ラグ）で外部知識を参照することで軽減できる。

使い方: 「ハルシネーションを減らすため、RAG（ラグ）で社内ドキュメントを参照し、根拠のある回答を生成するようにしました。」

レイテンシ（レイテンシ、Latency）

意味: 応答時間。リクエストからレスポンスまでの遅延。推論レイテンシはAIの回答速度。ユーザー体験に直結。

使い方: 「推論レイテンシを10ms未満に最適化し、リアルタイム性を確保しました。」

API（エーピーアイ、Application Programming Interface）

意味: ソフトウェア同士が連携するためのインターフェース。REST API（レストエーピーアイ）、OpenAI API（オープンエーアイエーピーアイ）など。外部サービスとの連携に使用。

使い方: 「OpenAI API（オープンエーアイエーピーアイ）と連携し、LLM（エルエルエム）の推論機能をアプリケーションに組み込みました。」
