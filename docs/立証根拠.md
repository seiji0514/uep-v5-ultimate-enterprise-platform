# 100システム 立証根拠

**目的**: 「100システム以上」の実績主張に対する立証・根拠をまとめたドキュメント。

---

## 1. カウントの定義

**「システム」の定義**: 独立したモジュール・API・マイクロサービス・画面・インフラコンポーネントのいずれかとして識別できる単位。

- バックエンドモジュール（ディレクトリ単位）
- Docker Compose で定義されたサービス
- プロジェクト・ディレクトリ（medical, space, traffic, projects, infrastructure, tools 配下）
- Chrome ブックマークから抽出したシステム名（設計・構築済みまたは構築中）

## 3. 立証方法

### 3.1 プロジェクト内からの抽出

- `backend/` 配下のモジュールディレクトリ
- `docker-compose*.yml` のサービス定義
- `medical/`, `space/`, `traffic/`, `projects/`, `infrastructure/`, `tools/` 配下のプロジェクト

### 3.2 Chrome ブックマーク

- `docs/chrome_bookmarks_システムのみ.md` から抽出
- 厳密モードで重複・外部サイト・求人・チュートリアルを除外
- システム名として識別可能なもののみカウント

### 3.3 参照ドキュメント

- `docs/100システム一覧_参照用.md` - 全抽出結果の統合
- `docs/100システム一覧_リスト.md` - リスト形式
- `docs/100システム一覧_公開用.md` - 公開用サマリ

## 4. 面談時の説明

**Q: 100システムの根拠は？**

> プロジェクト内のバックエンドモジュール、Docker サービス、医療・宇宙・交通等の専門プロジェクト、および Chrome ブックマークから抽出した設計・構築済みシステム名を合計しています。詳細は `docs/100システム一覧_参照用.md` で確認できます。デモ可能な UEP v5.0 内の機能も含まれます。
