# UEP v5.0 参画に必要なこと

**作成日**: 2026年3月

---

## 前提条件

| 項目       | 要件                       |
| ---------- | -------------------------- |
| Python     | 3.11以上                   |
| Node.js    | 18以上（フロントエンド用） |
| PostgreSQL | または SQLite              |
| Redis      | オプション                 |
| Docker     | 必須ではない               |

---

## 起動方法

### ローカル（Windows）

```cmd
start-all.bat
```

- バックエンド: http://localhost:8000
- フロントエンド: http://localhost:3000
- API ドキュメント: http://localhost:8000/docs

### ログイン後の見た目（変化の確認）

| ログイン前                 | ログイン後                                          |
| -------------------------- | --------------------------------------------------- |
| 中央にログインフォームのみ | **左にサイドバー**（メニュー一覧）                  |
| サイドバーなし             | **上部にヘッダー**（パンくず、ログアウト）          |
| -                          | **中央にダッシュボード**（MLOps、生成AI等のパネル） |

**デモ用ログイン**: `kaho0525` / `kaho052514` または `developer` / `dev123`

ログイン成功後、URL が `http://localhost:3000/` に変わり、サイドバーとダッシュボードが表示されます。

### クラウドデモ

- URL: https://uep-v5-ultimate-enterprise-platform.vercel.app
- Render の `/health` に事前アクセスでバックエンド起動

## 参画者が取り組めること

### システムの精度向上等は可能

| 領域           | 現状                                | 実装済み                                                                                        |
| -------------- | ----------------------------------- | ----------------------------------------------------------------------------------------------- |
| RAG検索精度    | ChromaDB、ベクトル検索              | ハイブリッド検索（ベクトル+キーワード）、RRF再ランキング、チャンク分割（`rag_enhancements.py`） |
| 推論品質       | reasoning_routing（タスク難度判定） | CoTプロンプト強化、難度別モデル選択（gpt-4/gpt-4o-mini/gpt-3.5-turbo）                          |
| 異常検知       | 簡易API                             | 閾値調整API、Z-score/ローリング特徴量、モデルアンサンブル（`core/anomaly_detector.py`）         |
| 稼働率         | 99.995%                             | ヘルスチェックリトライ、フェイルオーバー、サービスレジストリ（`core/health_failover.py`）       |
| 推論レイテンシ | <10ms                               | Redis/メモリキャッシュ、LLMフェイルオーバー（`llm_integration.py`）                             |

### その他

- **機能追加** - 新モジュール、API拡張、UI改善
- **品質強化** - テストカバレッジ、セキュリティ、監視の強化
- **ドキュメント整備** - 設計書、API仕様、運用マニュアル

実装済みの評価指標（RAG metrics、Prometheus、MLflow）を活用し、段階的な改善が可能です。
