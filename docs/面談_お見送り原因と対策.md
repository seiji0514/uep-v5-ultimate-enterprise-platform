# 書類審査通過後・面談でのお見送り 原因と対策

**作成日**: 2026年3月3日  
**目的**: 面談でお見送りになる主な原因と、その対策を整理する

---

## 1. お見送りの主な原因（想定）

### 1.1 実務経験・チーム開発に関する懸念

| 原因 | 詳細 |
|------|------|
| **実務経験の不足** | 個人開発が主で、実務（チーム開発・クライアント対応）の経験が少ないと見られる |
| **チーム開発経験の不足** | コードレビュー、ペアプログラミング、設計レビュー等の経験が少ない |
| **100システムの信憑性** | 「本当に動作するのか」「品質は？」と疑問を持たれる |

**対策**:
- `docs/EXPERIENCE_VS_SKILLS.md` の回答戦略を活用
- **実力・知識は実証済み**であることを具体的な数値で伝える（稼働率99.995%、推論レイテンシ<10ms）
- **チーム統括経験**（50名規模）と**リモート勤務実績**（約3年9ヶ月）を強調
- **デモで実物を見せる**ことで、実装力の信憑性を証明
- 「改善したいこと」として、チーム開発経験を積みたい旨を前向きに伝える

---

### 1.2 デモ・説明の失敗

| 原因 | 詳細 |
|------|------|
| **デモの不具合** | 起動失敗、403エラー、ログアウト等で実物を見せられない |
| **説明が長すぎる** | 相手の関心とずれ、時間を浪費する |
| **技術寄りすぎ** | 非エンジニアには伝わりにくい説明 |
| **デモの流れが不明瞭** | 何を見せているか相手に伝わらない |

**対策**:
- **事前にデモ環境を確認**（`docs/デモ_ボタンエラー対処.md` 参照）
- **トークン有効期限を延長**（`ACCESS_TOKEN_EXPIRE_MINUTES=120`）
- **代替手段を用意**（スクリーンショット・PDF）して、ライブデモが失敗した場合に備える
- **各パート1〜2分程度**に抑え、相手の反応を見て調整（`docs/面談用_デモ動画の流れ.md`）
- **チャット説明文**を事前に用意し、画面表示と同時に送る（`docs/面談用_ライブデモ_チャット説明文.txt`）

---

### 1.3 志望動機・入社意欲が伝わらない

| 原因 | 詳細 |
|------|------|
| **志望動機が曖昧** | 「なぜこの会社か」が伝わらない |
| **他社との差別化がない** | どこにでも使える汎用的な志望動機 |
| **入社意欲が弱く見える** | 熱量が伝わらない、受け身の印象 |

**対策**:
- 企業の**具体的な特徴**（制度・案件・文化）を調べ、志望動機に織り込む
- 「貴社の〇〇という点に惹かれた」と**具体的に**伝える
- 企業ごとに志望動機をカスタマイズ（`docs/履歴書_志望動機_*.md` 参照）

---

### 1.4 コミュニケーション・説明の課題

| 原因 | 詳細 |
|------|------|
| **説明が長い** | 1つの質問に1分以上答える、要点が伝わらない |
| **専門用語が多い** | 相手が理解しづらい |
| **質問の意図を外す** | 聞かれていないことを答える |
| **沈黙・間の悪さ** | 緊張で答えに詰まる |

**対策**:
- **結論ファースト**で答える（結論→理由→補足の順）
- **30秒〜1分**を目安に簡潔に答える
- 長く話す場合は「3点あります」など**構成を明示**
- 質問の意図が不明な場合は「〇〇についてお答えしてよろしいでしょうか」と確認
- `docs/INTERVIEW_ANSWERS_TEXT.md` で想定質問への回答を事前に準備

---

### 1.5 年齢・障害に関する懸念

| 原因 | 詳細 |
|------|------|
| **年齢の懸念** | 51歳で学習意欲・体力・キャリアの長さを心配される |
| **障害・配慮への不安** | 業務への支障、通院、入替え手術等を心配される |

**対策**:
- **年齢**: 超高速学習能力（IQ127、8ヶ月で100システム）と最新技術への対応力を具体的に伝える
- **障害**: 技術的作業に支障なし、完全リモートで健常者と同等以上のパフォーマンスを発揮できることを強調
- **障害説明書**を事前に提出し、配慮事項（通院の時間調整、完全リモート）を明確に記載
- 「決しててんかんではありません」等、誤解されやすい点は明確に説明

---

### 1.6 その他の原因

| 原因 | 詳細 |
|------|------|
| **希望年収のミスマッチ** | 企業の想定を大きく上回る |
| **案件・条件のミスマッチ** | 希望する案件・勤務形態が合わない |
| **相性・フィット感** | 雰囲気や価値観が合わないと判断される |
| **他候補との比較** | より条件の良い候補がいた |

**対策**:
- 希望年収は「貴社の規定に合わせて柔軟に」と伝え、成長機会を最優先する姿勢を示す
- 企業の制度（還元率、前職給与保証等）に言及し、理解していることを示す
- 相性はコントロールできない部分もあるため、**複数社に並行して応募**し、選択肢を広げる

## 2. 面談前チェックリスト（対策の実践）

### 2.1 事前準備

| 項目 | 確認 |
|------|------|
| デモ環境の起動確認 | バックエンド・フロントエンドが正常に起動するか |
| ログイン情報の確認 | kaho0525 / kaho052514 でログインできるか |
| トークン有効期限 | 120分に設定済みか（ログアウト防止） |
| 代替手段の準備 | スクリーンショット・PDFを用意しているか |
| チャット説明文 | `面談用_ライブデモ_チャット説明文.txt` を開いているか |

### 2.2 回答準備

| 項目 | 確認 |
|------|------|
| 志望動機 | 企業ごとにカスタマイズした志望動機を準備しているか |
| 実務経験への回答 | 「実力・知識は実証済み」「チーム経験は部分的に習得済み」の流れで説明できるか |
| 長所・短所 | 簡潔に答えられるか（`docs/長所・短所（面談用）.txt`） |
| 希望年収 | 柔軟に対応する姿勢を伝えられるか |

### 2.3 面談中の心がけ

| 項目 | 心がけ |
|------|--------|
| 説明の長さ | 1つの回答は30秒〜1分程度に抑える |
| デモの長さ | 各パート1〜2分、相手の反応を見て調整 |
| 結論ファースト | 結論→理由→補足の順で答える |
| 相手の反応 | 相手が理解しているか、興味を持っているかを確認する |

## 3. お見送り後の振り返り

お見送りになった場合、可能であれば**フィードバックを依頼**する。

- 「今後の改善のため、ご意見をいただけますと幸いです」
- フィードバックが得られれば、次回の面談に活かせる

得られない場合でも、**自己振り返り**を行う：

- デモはスムーズにできたか
- 説明は簡潔だったか
- 志望動機は伝わったか
- 質問への回答で詰まった箇所はなかったか
