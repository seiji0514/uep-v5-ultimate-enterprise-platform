# 転職ドラフトスカウト - 個人プロジェクトのレジュメ記載方法

**作成日**: 2026年1月30日  
**目的**: 就活中の個人プロジェクトをレジュメに記載する方法

---

## 📋 状況整理

### ユーザーの状況

- **期間**: 令和6年8月（2024年8月）から現時点まで
- **状況**: 就活中
- **プロジェクト**: 個人プロジェクト（会社のプロジェクトではない）
- **役割**: 全ての工程を自分で行った

---

## ✅ 個人プロジェクトの記載方法

### 重要なポイント

**個人プロジェクトでも、適切に記載すれば審査通過可能**

- ✅ **個人プロジェクトも有効**: 会社のプロジェクトでなくても、個人プロジェクトは有効な経験として評価される
- ✅ **全ての工程を記載**: 設計、実装、テスト、デプロイなど、全ての工程を記載
- ✅ **具体的に記載**: 技術スタック、成果、規模などを具体的に記載

- コンテナ: Docker, Docker Compose
- IaC: Terraform
- CI/CD: GitHub Actions
```

- 通常の見積もりの約180-270倍の効率で開発（エンタープライズレベルを1日で実装）

【プロジェクト規模】
- 個人開発
- 実装時間: 約1日（エンタープライズレベル）
- コード行数: 10,000+行（バックエンド + フロントエンド）
- APIエンドポイント数: 50+エンドポイント
```

#### 規模

## 📋 全ての工程を記載する方法

### 工程別の記載例

#### 1. 設計工程

```
【設計工程】
- アーキテクチャ設計: マイクロサービスアーキテクチャ、イベント駆動設計
- データベース設計: PostgreSQL、Redis、データモデル設計
- API設計: RESTful API、OpenAPI仕様、Swagger UI
- セキュリティ設計: JWT認証、RBAC/ABAC、Zero Trust、mTLS
- UI/UX設計: Material-UI、レスポンシブデザイン、アクセシビリティ対応
```

#### 2. 実装工程

```
【実装工程】
- バックエンド実装: FastAPI、Python 3.11+、7つのコアシステムの実装
- フロントエンド実装: React 18+、TypeScript、Material-UI、8ページの実装
- インフラ実装: Docker、Docker Compose、Kubernetes、Terraform
- 認証・認可実装: JWT、OAuth2/OIDC、RBAC/ABAC
- API Gateway実装: Kong、Envoy
- データレイク実装: MinIO
- イベントストリーミング実装: Apache Kafka
- 監視・オブザーバビリティ実装: Prometheus、Grafana、ELK Stack
```

#### 3. テスト工程

```
【テスト工程】
- 単体テスト: pytest、Jest
- 統合テスト: システム間の統合テスト
- E2Eテスト: フロントエンド・バックエンドの統合テスト
- パフォーマンステスト: 負荷テスト、スケーラビリティテスト
- セキュリティテスト: セキュリティ監査、脆弱性スキャン
```

#### 4. デプロイ・運用工程

```
【デプロイ・運用工程】
- CI/CDパイプライン: GitHub Actions
- コンテナ化: Docker、Docker Compose
- オーケストレーション: Kubernetes
- インフラコード: Terraform
- モニタリング: Prometheus、Grafana
- ログ管理: ELK Stack
- エラーハンドリング: 詳細なエラーメッセージ、ログ記録
```

#### 5. ドキュメント工程

```
【ドキュメント工程】
- APIドキュメント: Swagger UI、OpenAPI仕様
- 技術ドキュメント: アーキテクチャドキュメント、実装ガイド
- ユーザーガイド: デモンストレーションガイド、クイックスタートガイド
- トラブルシューティングガイド: エラー対処法、FAQ
```

## 💡 記載のポイント

### ポイント1: 個人プロジェクトであることを明記

**「個人開発」「個人プロジェクト」と明記**

- ✅ 「個人開発・就活中」と記載
- ✅ 「個人プロジェクト」と記載
- ✅ 「全ての工程を自分で担当」と記載

### ポイント2: 全ての工程を具体的に記載

**設計、実装、テスト、デプロイ、運用、ドキュメントなど**

- ✅ 各工程で使用した技術・ツールを記載
- ✅ 各工程での成果を記載
- ✅ 各工程での課題と解決方法を記載

### ポイント3: 技術スタックを詳細に記載

**使用した技術・ツールを具体的に記載**

- ✅ プログラミング言語、フレームワーク
- ✅ データベース、インフラ
- ✅ 開発ツール、CI/CD
- ✅ 監視・ログ管理ツール

### ポイント4: 成果を具体的に記載

**数値や具体的な成果を記載**

- ✅ 実装したシステム数、機能数
- ✅ コード行数、APIエンドポイント数
- ✅ 開発効率（通常の見積もりとの比較）
- ✅ 技術的成果、学習成果

### ポイント5: 規模を具体的に記載

**プロジェクトの規模を具体的に記載**

- ✅ 開発体制（個人開発）
- ✅ 実装期間
- ✅ 実装時間
- ✅ コード行数、APIエンドポイント数
