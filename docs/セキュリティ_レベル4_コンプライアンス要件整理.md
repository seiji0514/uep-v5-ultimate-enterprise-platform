# コンプライアンス要件の整理

**作成日**: 2026年3月2日  
**対象**: UEP v5.0 サイバー対策基盤（レベル4）

---

## 1. 概要

本ドキュメントは、UEP サイバー対策基盤が対応すべき主要なコンプライアンス要件を整理する。実装・運用の指針として参照する。

---

## 2. 主要フレームワーク・規制

### 2.1 一覧

| フレームワーク | 対象 | 主な要件 |
|----------------|------|----------|
| **ISO 27001** | 情報セキュリティマネジメント | ISMS、リスク評価、監査 |
| **SOC 2** | サービス組織 | セキュリティ、可用性、機密性 |
| **NIST CSF** | サイバーセキュリティ | Identify, Protect, Detect, Respond, Recover |
| **PCI DSS** | カード決済 | カードデータ保護、アクセス制御 |
| **GDPR** | 個人データ（EU） | 同意、削除権、データポータビリティ |
| **個人情報保護法** | 個人データ（日本） | 利用目的、安全管理措置 |

---

## 3. 要件別整理

### 3.1 ログ・監査

| 要件 | 内容 | UEP 対応 |
|------|------|----------|
| **監査ログの記録** | 重要な操作のログ記録 | Security Center イベント、アクセスログ |
| **ログの改ざん防止** | 追記のみ、削除不可 | ファイルパーミッション、WORM ストレージ |
| **ログ保持期間** | 規制に応じた保持 | 90日〜1年（設定可能） |
| **ログの検索** | 監査時の検索 | SIEM 検索 API | 

### 3.2 アクセス制御

| 要件 | 内容 | UEP 対応 |
|------|------|----------|
| **最小権限** | 必要最小限の権限付与 | RBAC |
| **認証** | 強パスワード、MFA | JWT、OAuth2（MFA は拡張可能） |
| **セッション管理** | タイムアウト、不正利用検知 | セッションタイムアウト警告 |
| **権限の定期確認** | 権限の棚卸し | 管理者による手動確認 |

### 3.3 インシデント対応

| 要件 | 内容 | UEP 対応 |
|------|------|----------|
| **インシデント検知** | 検知・記録 | Security Center、Suricata、Wazuh |
| **対応手順** | 手順書の整備 | 本番運用手順.md |
| **対応記録** | 対応内容の記録 | インシデント API |
| **事後分析** | 再発防止 | コンプライアンスレポート |

### 3.4 データ保護

| 要件 | 内容 | UEP 対応 |
|------|------|----------|
| **暗号化（保存時）** | データベース暗号化 | PostgreSQL TDE（要設定） |
| **暗号化（通信時）** | TLS | HTTPS、mTLS |
| **脆弱性管理** | 定期的なスキャン | pip-audit、OWASP ZAP |
| **パッチ適用** | セキュリティパッチの適用 | 運用手順 |

### 3.5 リスク管理

| 要件 | 内容 | UEP 対応 |
|------|------|----------|
| **リスク評価** | 定期的なリスク評価 | リスク分析 API |
| **リスク対応** | 軽減策の実施 | リスク登録・管理 |
| **セキュリティ態勢** | 全体の把握 | security-posture API |

## 4. フレームワーク別マッピング

### 4.1 NIST CSF

| 機能 | Identify | Protect | Detect | Respond | Recover |
|------|----------|----------|--------|----------|---------|
| **UEP 機能** | リスク分析、資産管理 | Zero Trust、mTLS、RBAC | Suricata、Wazuh、SIEM | インシデント対応、自動対応 | コンプライアンスレポート | 

### 4.2 ISO 27001  Annex A

| 管理策 | 内容 | UEP 対応 |
|--------|------|----------|
| A.12.4.1 イベントログ | ログの記録 | セキュリティイベント、監査ログ |
| A.12.4.2 管理者ログ | 特権操作の記録 | 認証・認可ログ |
| A.12.4.3 クロック同期 | 時刻同期 | NTP 設定（運用） |
| A.12.6.1 技術的脆弱性 | 脆弱性管理 | 脆弱性スキャン | 

### 4.3 SOC 2 Trust Criteria

| 基準 | 内容 | UEP 対応 |
|------|------|----------|
| **CC6.1** 論理的・物理的アクセス制御 | アクセス制御 | RBAC、認証 |
| **CC6.6** 論理的・物理的アクセス制御の監視 | 監視 | セキュリティイベント、アラート |
| **CC7.1** システムオペレーション | 変更管理 | デプロイ手順 |
| **CC7.2** 異常検知 | 異常検知 | 異常検知設計 |
| **CC7.3** インシデント対応 | 対応 | インシデント API、運用手順 |

## 5. コンプライアンスレポート項目

UEP の `/api/v1/cyber-defense/compliance/report` が出力する項目と、対応する要件:

| レポート項目 | 対応要件 |
|--------------|----------|
| 総イベント数 | 監査証跡 |
| Critical/High 件数 | インシデント検知 |
| アクセスログサマリ | アクセス制御の監視 |
| 推奨事項 | 継続的改善 |

### 5.1 拡張項目（将来）

| 項目 | 説明 |
|------|------|
| ログ保持期間 | 設定値と実績 |
| 脆弱性スキャン結果 | 直近のスキャン日時・件数 |
| インシデント対応率 | 対応完了 / 総件数 |
| アクセス権限一覧 | ユーザー別権限 |
