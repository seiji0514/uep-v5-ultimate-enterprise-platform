# バックアップ手順

UEP v5.0 + 個人会計 + docs を外付けUSB 2台・Google Drive・外付けドライブへバックアップする手順です。

---

## 保存対象（uep-v5-ultimate-enterprise-platform 配下）

| システム | フォルダ |
|----------|----------|
| UEP v5.0 | backend, frontend |
| 統合セキュリティ・防衛プラットフォーム | security_defense_platform |
| 製造・IoTプラットフォーム | manufacturing_platform |
| 医療・ヘルスケアプラットフォーム | medical_platform |
| 金融・FinTechプラットフォーム | fintech_platform |
| 産業統合プラットフォーム | industry_unified_platform |
| 企業横断オペレーション基盤 | enterprise_operations_hub |
| 個人会計（UEP版） | backend/personal_accounting, frontend の PersonalAccounting |
| 個人用 PC 容量確保 | personal-pc-cleanup |

バックアップ実行後、各先に **バックアップ内容一覧.txt** が自動作成されます。

---

## 一括バックアップ（推奨）

**外付けUSB 2台 + Google Drive + 外付けドライブ** へ一括コピーします。

```cmd
cd C:\uep-v5-ultimate-enterprise-platform
scripts\一括バックアップ_全先.bat
```

### ドライブ設定

`scripts\一括バックアップ_全先.bat` の上部でドライブレターを変更できます。

| 変数 | デフォルト | 説明 |
|------|------------|------|
| USB1 | N: | 外付けドライブ N（ELECOM USBHDD 等） |
| USB2 | O: | Google Drive O のマイドライブ |
| GDRIVE | （空） | 別の Google Drive パス（空なら [3/4] スキップ） |
| EXT_DRIVE | E: | その他外付けドライブ（使わない場合は空に） |

### 自動実行（タスクスケジューラ）

日次バックアップを自動化する場合:

1. タスクスケジューラで「基本タスクの作成」
2. トリガー: 毎日（例: 22:00）
3. 操作: `scripts\一括バックアップ_全先.bat`、引数 `auto`

---

## USB N と O へ追加・更新（既存フォルダを更新）

3/7 等で作成済みの uep-v5-backup に、変更分を上書き・新規を追加します。日付フォルダは作成しません。

```cmd
scripts\一括バックアップ_更新のみ_USB_N_O.bat
```

- **日付フォルダを更新する場合**: スクリプト内の `set "USE_DATE=2026-03-07"` を設定
- **直下を更新する場合**: `USE_DATE` を空のまま（デフォルト）

---

## 外付けドライブ N と Google Drive O へ追加・更新

USB N と O と同じ様に、既存 uep-v5-backup へ追加・更新します。

```cmd
scripts\一括バックアップ_更新のみ_N_GoogleDrive.bat
```

- **外付けドライブ N**: N:（ELECOM USBHDD 等）
- **Google Drive O**: O:\（O: が Google Drive のマイドライブの場合）

---

## 個別バックアップ

| スクリプト | 先 | 用途 |
|------------|-----|------|
| `scripts\外付けUSB1_バックアップ_N.bat` | N: | USB 1 のみ（日付フォルダ作成） |
| `scripts\外付けUSB2_バックアップ_O.bat` | O: | USB 2 のみ（日付フォルダ作成） |
| `scripts\GoogleDrive_バックアップ.bat` | N:\マイドライブ | Google Drive のみ |

## コピー対象

- UEP v5.0 本体（node_modules, venv, __pycache__, .git 除外）
- docs（職務経歴書・ポートフォリオ等）
