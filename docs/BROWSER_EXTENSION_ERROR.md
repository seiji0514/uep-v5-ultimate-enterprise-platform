# ブラウザ拡張機能エラー対処ガイド

**作成日**: 2026年1月30日

---

## 問題の症状

ブラウザのコンソールに以下のエラーが表示される：

```
Uncaught (in promise) The message port closed before a response was received.
```

エラーのソース: `content.js:1`

---

## 原因

このエラーは、**ブラウザ拡張機能**（Chrome拡張機能）が原因です。アプリケーション自体の問題ではありません。

`content.js`は通常、ブラウザ拡張機能のコンテンツスクリプトで、以下のような拡張機能が原因の可能性があります：

- Cursor拡張機能
- 開発ツール拡張機能
- その他のChrome拡張機能

---

## 対処方法

### 方法1: 拡張機能を無効化して確認（推奨）

1. **拡張機能を無効化**
   - Chromeの設定 → 拡張機能
   - 最近インストールした拡張機能を一時的に無効化

2. **ページを再読み込み**
   - エラーが消えるか確認

3. **原因の拡張機能を特定**
   - 拡張機能を1つずつ有効化して、エラーが再現する拡張機能を特定

### 方法2: シークレットモードで確認

1. **シークレットモードで開く**
   - `Ctrl + Shift + N` (Windows)
   - 拡張機能が無効な状態で動作確認

2. **エラーが消えるか確認**
   - シークレットモードでエラーが消えれば、拡張機能が原因です

### 方法3: エラーを無視する（推奨）

このエラーは**アプリケーションの動作には影響しません**。

- ブラウザ拡張機能の内部エラーです
- アプリケーションのAPI呼び出しや機能には影響しません
- 無視しても問題ありません

---

## 確認方法

### アプリケーションが正常に動作しているか確認

1. **Swagger UIでAPIをテスト**
   - `http://localhost:8000/docs`
   - APIエンドポイントを実行して、正常に動作するか確認

2. **フロントエンドでログイン**
   - `http://localhost:3000`
   - ログインして、データが取得できるか確認

3. **ネットワークタブを確認**
   - 開発者ツール → Networkタブ
   - APIリクエストが正常に送信されているか確認
   - ステータスコードが200や201であれば問題ありません

## よくある質問

### Q: このエラーは修正する必要がありますか？

**A:** いいえ。このエラーはブラウザ拡張機能の内部エラーで、アプリケーションの動作には影響しません。

### Q: エラーが表示され続けますが、問題ありませんか？

**A:** はい、問題ありません。アプリケーションが正常に動作していれば、このエラーは無視して構いません。

### Q: エラーを完全に消したい場合は？

**A:** 原因となっているブラウザ拡張機能を無効化するか、シークレットモードで使用してください。
