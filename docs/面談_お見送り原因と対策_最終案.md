# 書類審査通過後・面談でのお見送り 原因と対策【最終案】

**作成日**: 2026年3月3日  
**用途**: 面談前の確認・面談中の心がけ・お見送り後の振り返り

---

## ■ 面談直前 実行要点（30秒で確認）

| 項目 | やること |
|------|----------|
| **説明** | 結論ファースト、30秒〜1分で答える |
| **デモ** | 各パート1〜2分、事前に起動確認 |
| **実務経験** | 「実力は実証済み」「チーム経験は部分的に習得済み」「改善意欲あり」 |
| **志望動機** | 企業の具体的な特徴を1つ以上織り込む |
| **年齢・障害** | 「支障なし」「リモートで同等以上」を簡潔に |

---

## 1. お見送りの主な原因と対策

### 1.1 実務経験・チーム開発

| 原因 | 対策 |
|------|------|
| 実務経験の不足 | 稼働率99.995%、推論レイテンシ<10ms 等の数値で実証済みを伝える |
| チーム開発経験の不足 | 50名統括・3年9ヶ月リモート実績を強調。「改善したいこと」として前向きに |
| 100システムの信憑性 | **デモで実物を見せる**。動作するシステムであることを証明 |

**回答の流れ**（`EXPERIENCE_VS_SKILLS.md` 参照）:
1. 実力・知識は実証済み（30秒）
2. チーム経験は部分的に習得済み（30秒）
3. 改善意欲を示す（30秒）

---

### 1.2 デモ・説明の失敗

| 原因 | 対策 |
|------|------|
| デモの不具合 | 事前に起動確認。トークン120分に延長。代替（スクショ・PDF）を用意 |
| 説明が長い | 各パート1〜2分。チャット説明文をコピペで補足 |
| 流れが不明瞭 | `面談用_デモ動画の流れ.md` の順序で実施 |

---

### 1.3 志望動機・入社意欲

| 原因 | 対策 |
|------|------|
| 志望動機が曖昧 | 「貴社の〇〇（制度・案件・文化）に惹かれた」と具体的に |
| 他社との差別化なし | 企業ごとに `履歴書_志望動機_*.md` でカスタマイズ |

---

### 1.4 コミュニケーション

| 原因 | 対策 |
|------|------|
| 説明が長い | **結論→理由→補足**。30秒〜1分を目安 |
| 専門用語が多い | 相手の反応を確認。必要なら「〇〇という技術で」と補足 |
| 質問の意図を外す | 不明なら「〇〇についてでよろしいでしょうか」と確認 |

---

### 1.5 年齢・障害

| 原因 | 対策 |
|------|------|
| 年齢の懸念 | IQ127、8ヶ月で100システム。学習意欲・対応力を数値で伝える |
| 障害への不安 | 技術的作業に支障なし。完全リモートで同等以上。障害説明書を事前提出 |

---

### 1.6 その他

| 原因 | 対策 |
|------|------|
| 希望年収のミスマッチ | 「貴社の規定に合わせて柔軟に」「成長機会を最優先」 |
| 相性・他候補 | コントロール不可。複数社に並行応募で選択肢を広げる |
