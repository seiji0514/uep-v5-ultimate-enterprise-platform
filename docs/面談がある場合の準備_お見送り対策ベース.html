1. 面談直前 実行要点（30秒で確認）

項目 やること
説明 結論ファースト、30秒〜1分で答える
デモ 各パート1〜2分、事前に起動確認
実務経験 「実力は実証済み」「チーム経験は部分的に習得済み」「改善意欲あり」
志望動機 企業の具体的な特徴を1つ以上織り込む
年齢・障害 「支障なし」「リモートで同等以上」を簡潔に

2. タイムライン別 やること

2.1 面談決定時（面談日が決まったら）

  • 本ファイルの「面談決定時に記入」を記入
  • 志望動機を企業ごとにカスタマイズ
  • 面談日時をカレンダーに登録
  • デモ環境の起動確認（1回試す）

2.2 面談前日

デモ準備

  • バックエンド・フロントエンド起動確認
  • ログイン（kaho0525 / kaho052514）確認
  • トークン有効期限120分に設定
  • 代替手段（スクショ・PDF）用意
  • 面談用_ライブデモ_チャット説明文.txt を開く

回答準備

  • 志望動機を声に出して練習
  • 実務経験への回答（3ステップ）を練習
  • 長所・短所を確認
  • 希望年収の伝え方

2.3 面談当日（開始10分前）

  • デモ環境を起動
  • ログイン済みであることを確認
  • チャット説明文を開く
  • 実行要点を30秒で確認
  • 挨拶・締めの言葉を確認

2.4 面談中

  • 1回答＝30秒〜1分
  • デモ各パート＝1〜2分
  • 結論ファースト
  • 相手の反応を確認

2.5 面談後

  • お礼メール送信（24時間以内）
  • 振り返りメモを記録

3. 挨拶・締めの言葉

冒頭の挨拶

「本日はお時間をいただき、ありがとうございます。面談の機会をいただき、大変光栄に思います。少し緊張しておりますが、ご了承ください。」

締めの言葉

「本日は、貴重なお時間をいただき、誠にありがとうございました。今後の選考の流れについて、お教えいただけますでしょうか？何か追加でご質問がございましたら、お気軽にお聞かせください。本日は、ありがとうございました。」

5. お見送り原因と対策（クイック参照）

原因カテゴリ 対策の一言
実務経験の不足 稼働率99.995%、推論<10ms で実証済みを伝える
チーム開発経験 50名統括・3年9ヶ月リモート。「改善したいこと」として前向きに
100システムの信憑性 デモで実物を見せる
デモの不具合 事前起動確認。トークン120分。代替（スクショ・PDF）
志望動機が曖昧 「貴社の〇〇に惹かれた」と具体的に
説明が長い 結論→理由→補足。30秒〜1分
年齢の懸念 IQ127、8ヶ月で100システム。数値で伝える
障害への不安 支障なし。完全リモートで同等以上。障害説明書を事前提出
希望年収 「貴社の規定に合わせて柔軟に」「成長機会を最優先」

6. お見送り後の振り返り

  • フィードバック依頼: 「今後の改善のため、ご意見をいただけますと幸いです」
  • 自己振り返り: デモ・説明・志望動機・回答で詰まった箇所を記録
