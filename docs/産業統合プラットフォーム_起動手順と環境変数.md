# 産業統合プラットフォーム - 起動手順と環境変数

**作成日**: 2026年3月15日  
**対象**: 産業統合プラットフォーム（製造・IoT / 医療・金融・セキュリティ等 8ドメイン統合）

---

## 起動手順

### 産業統合プラットフォーム（単独起動）

1. **バックエンド起動**
   ```powershell
   cd industry_unified_platform
   .\start-backend.bat
   ```
   または手動:
   ```powershell
   cd backend
   venv\Scripts\activate
   python ..\industry_unified_platform\main.py
   ```
   ※ポート9010で起動（使用中なら9011〜9015を自動選択）

2. **フロントエンド起動**
   ```powershell
   cd industry_unified_platform
   .\start-frontend.bat
   ```
   または手動:
   ```powershell
   cd industry_unified_platform\frontend
   npm install   # 初回のみ
   npm start
   ```

3. **アクセス**
   - フロント: http://localhost:3000
   - バックエンドAPI: http://localhost:9010

---

### UEP v5.0 統合基盤（手動起動）

1. **バックエンド**
   ```powershell
   cd backend
   venv\Scripts\activate
   python main.py
   ```
   ※ポート8000で起動

2. **フロントエンド**
   ```powershell
   cd frontend
   npm install   # 初回のみ
   npm start
   ```

3. **アクセス**
   - フロント: http://localhost:3000
   - API: http://localhost:8000

## 環境変数

### 産業統合プラットフォーム フロントエンド

| 変数名 | 説明 | デフォルト |
|--------|------|------------|
| `REACT_APP_INDUSTRY_API_URL` | 産業統合APIのベースURL | http://localhost:9010 |
| `REACT_APP_INDUSTRY_UNIFIED_URL` | 上記の別名（互換用） | 同上 |

### 産業統合プラットフォーム バックエンド

| 変数名 | 説明 | デフォルト |
|--------|------|------------|
| `ENVIRONMENT` | 実行環境（production / development） | production |
| `DEBUG` | デバッグモード（true/false） | false |
| `INDUSTRY_UNIFIED_PORT` | 待ち受けポート（0=自動） | 0 |
| `INDUSTRY_UNIFIED_HOST` | 待ち受けホスト | 127.0.0.1 |
| `CORS_ORIGINS` | CORS許可オリジン（カンマ区切り） | * |

### UEP v5.0 共通

| 変数名 | 説明 | デフォルト |
|--------|------|------------|
| `DATABASE_URL` | DB接続URL | sqlite:///./uep_db.sqlite |
| `REDIS_URL` | Redis接続URL（オプション） | - |
| `SECRET_KEY` | JWT署名用シークレット | - |

---

## 起動方法の違い

| システム | 推奨起動 | 備考 |
|----------|----------|------|
| 産業統合 | `.bat` または手動 | ポート9010、UEPとは別プロセス |
| UEP v5.0 | 手動 | ポート8000、統合基盤 |

産業統合とUEP v5.0は**同時起動可能**（別ポートのため）。
