## デモの進め方（面談次第で実施する場合）

デモは面談の流れに合わせて実施する。事前に環境を起動し、手順書を手元に用意しておく。

**実績・プロジェクトを聞かれたとき**
「実際に動作しているシステムがあります。画面共有でデモンストレーションできます」と伝え、了承されたら画面共有でデモを行う。

**技術的な深掘りをされたとき**
UEP v5.0 を起動し、画面共有でデモを行う。Swagger UI とフロントのダッシュボードを中心に説明する。

**時間が足りない／話題が別のとき**
無理にデモはせず、口頭で概要（7つのコアシステム、技術スタック、品質指標など）を説明する。

**面接官から「デモを見せてください」と言われたとき**
すぐに画面共有に切り替え、本手順書の「面談当日の起動手順」以降に従ってデモを行う。

**事前にしておくこと**
UEP v5.0 を起動できる状態にしておく（start-all.bat 等）。本手順書（DEMO_FOR_INTERVIEW.md）を開いておく。Zoom の画面共有の操作を確認しておく。

よくある質問（FAQ）

Q1: なぜWindows環境で実行するようにしたのですか？

A: デモンストレーション環境として、WindowsデスクトップPCでの実行を想定していたためです。WSLやDocker Desktopが不要で、より簡単に起動できるようにしました。

Q2: 認証情報は変更できますか？

A: はい、backend/auth/routes.pyの_init_demo_users`関数で変更できます。現在は`kaho0525` / `kaho052514`に設定されています。

Q3: 本番環境での使用は可能ですか？

A: 現在の実装は開発・デモンストレーション用です。本番環境で使用する場合は、以下の点を確認・修正してください：
- セキュリティ設定の見直し
- データベースの設定（現在はメモリ内ストレージ）
- 環境変数の適切な設定
- ログとモニタリングの設定

Q4: パフォーマンスはどの程度ですか？

A: 現在の実装は開発環境向けです。本番環境では、以下の最適化を推奨します：
- データベースの最適化
- キャッシング戦略の実装
- ロードバランシングの設定
- CDNの使用

トラブル時の対応手順

問題1: バックエンドが起動しない

症状: `start-backend.bat`を実行してもエラーが発生

対応手順:
1. Pythonがインストールされているか確認: `python --version`
2. 仮想環境を再作成:
   cmd
   cd backend
   rmdir /s /q venv
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```
3. それでも起動しない場合:
   - 「バックエンドの起動に問題が発生しましたが、Swagger UIのスクリーンショットでAPI仕様をご確認いただけます」と説明
   - 事前に準備したスクリーンショットを共有

---

問題2: フロントエンドが起動しない

症状: start-frontend.bat を実行してもエラーが発生

対応手順:
1. Node.jsがインストールされているか確認: `node --version`
2. 依存関係を再インストール:
   ```cmd
   cd frontend
   rmdir /s /q node_modules
   npm install
   npm start
   ```
3. それでも起動しない場合:
   - 「フロントエンドの起動に問題が発生しましたが、バックエンドAPIの動作をご確認いただけます」と説明
   - Swagger UIでAPIの動作を実演

---

問題3: ログインできない

症状: ログイン画面でエラーが発生

対応手順:
1. バックエンドが起動しているか確認: `http://localhost:8000/health`
2. ユーザー名とパスワードが正しいか確認:
   - ユーザー名: `kaho0525`
   - パスワード: `kaho052514`
3. ブラウザのコンソールでエラーを確認
4. それでもログインできない場合:
   - 「ログイン機能に問題が発生しましたが、Swagger UIでAPIの動作をご確認いただけます」と説明
   - Swagger UIでAPIの動作を実演

---

問題4: 画面共有ができない

症状: Google Meetの画面共有ができない

対応手順:
1. ブラウザのウィンドウを事前に開いておく
2. 画面共有の代わりに、事前に準備したスクリーンショットを共有
3. または、コードを直接共有して説明
