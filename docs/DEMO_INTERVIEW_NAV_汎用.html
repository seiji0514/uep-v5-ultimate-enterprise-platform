デモで使うAI用語集（聞かれたときの説明用）

面談で「〇〇とは？」と聞かれたとき、簡潔に説明できるようピックアップ。

RAG（ラグ、Retrieval-Augmented Generation）

意味: 検索拡張生成。外部知識を検索してLLMの入力に加え、根拠のある回答を生成。ハルシネーション軽減。

デモでの説明: 「生成AIのRAGタブで、ドキュメント検索とLLMによる回答生成の連携をご確認いただけます。」

CoT（コット、Chain-of-Thought）推論

意味: 思考の連鎖。段階的に推論してから結論を出す手法。複雑な問題で精度向上。

デモでの説明: 「CoT推論をプロンプトに組み込み、論理的な回答を生成するように実装しています。」

MLOps / LLMOps

意味: MLモデル／LLMの運用管理。実験管理、モデルレジストリ、デプロイ、監視。

デモでの説明: 「MLOpsタブでパイプライン・モデル・実験管理、Prometheus/Grafanaで稼働率99.995%を監視しています。」

インクルーシブ雇用AI

意味: 障害者雇用マッチング、アクセシビリティ特化AI、当事者視点UX評価を統合したプラットフォーム。

デモでの説明: 「サイドメニューのインクルーシブ雇用AIで、求人マッチング・アクセシビリティAI・UX評価をご確認いただけます。」

Chaos Engineering / GraphQL / Contract Testing

意味: Chaos=障害シミュレーションでレジリエンス検証。GraphQL=柔軟なクエリ。Contract Testing=API契約のテスト保証。

デモでの説明: 「Chaos Engineeringで遅延・エラー注入、GraphQLで柔軟なクエリ、Contract TestingでAPI品質を保証しています。」

※その他用語は docs/DEMO_INTERVIEW_NAV_RASUKU.html を参照

デモの進め方

デモは面談の流れに合わせて実施する。事前に環境を起動し、手順書を手元に用意しておく。

  • 実績・プロジェクトを聞かれたとき 「実際に動作しているシステムがあります。画面共有でデモンストレーションできます」と伝え、了承されたら画面共有でデモを行う。
  • 技術的な深掘りをされたとき UEP v5.0 を起動し、Swagger UIとフロントのダッシュボードを中心に説明する。
  • AI用語の意味を聞かれたとき 上記「デモで使うAI用語集」を参照し、簡潔に説明してから実装例を補足。
  • 時間が足りない／話題が別のとき 無理にデモはせず、口頭で概要（Level 1～5 統合、技術スタック、品質指標など）を説明する。
  • 事前にしておくこと start-backend.bat、フロントエンド（cd frontend → npm start）で起動できる状態にしておく。本手順書を開いておく。Zoom/Meetの画面共有を確認しておく。

  1. ログイン → http://localhost:3000 で kaho0525 / kaho052514 または developer / dev123 でログイン

実務経験: KDDIエボルバ（11ヶ月）、AI技術研究開発（8ヶ月・100システム以上）、JCI（2年2ヶ月・Webアクセシビリティ検査）。

トラブル時の対応

バックエンドが起動しない: ポート8000が使用中の場合は8001で起動。それでもダメなら Swagger UI のスクリーンショットで説明。

フロントエンドが起動しない: node_modules 削除 → npm install。それでもダメなら Swagger UI でAPI実演。

ログインできない: バックエンド起動確認 → 認証情報確認。それでもダメなら Swagger UI でAPI実演。
