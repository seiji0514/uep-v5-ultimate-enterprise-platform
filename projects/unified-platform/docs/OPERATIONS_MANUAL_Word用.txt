統合基盤 運用マニュアル

※ 本システムは専門システムであり、デモ向きではない。UEP v5.0 は別のデモ用システム。


1. 起動・停止

起動

cd /mnt/c/uep-v5-ultimate-enterprise-platform/projects/unified-platform
chmod +x start.sh   （初回のみ）
./start.sh


アクセス

URL: http://localhost:8000
ログイン: シードで作成された kaho0525/0525 または admin/admin（デフォルトは DB 認証のみ）
※ DEMO_LOGIN_ENABLED=true にするとデモ認証も有効（専門システムでは非推奨）


localhost が使えない場合（WSL）

hostname -I | awk '{print $1}'

表示された IP で http://<IP>:8000 にアクセス。WSL 再起動で IP が変わる場合あり。


ポート競合時

WSL 内に PostgreSQL/Redis がある場合、Docker 起動前に実行:

sudo service postgresql stop
sudo service redis-server stop

古いコンテナ: docker compose down --remove-orphans で削除。


停止

docker compose down


2. トラブルシューティング

  状況                    対処
  接続拒否                1〜2分待って再試行。docker compose logs app でエラー確認
  ログインできない        docker compose up -d --build で再ビルド
  白い画面                WSL IP でアクセス（hostname -I）
  5432 address in use      WSL 内 PostgreSQL 停止: sudo service postgresql stop
                          または docker compose の db は 5433 に変更済み（競合回避）
  6379 address in use      WSL 内 Redis 停止: sudo service redis-server stop
                          または docker compose の redis は 6380 に変更済み（競合回避）


3. ログ確認

docker compose logs -f app


4. シードデータ再投入

curl -X POST http://localhost:8000/api/v1/seed

またはブラウザで http://localhost:8000/api/v1/seed を開く。


5. DB バックアップ

docker compose exec db pg_dump -U unified unified > backup_$(date +%Y%m%d).sql

バックアップ自動化（cron）:
毎日 2:00 にバックアップ（WSL 内で crontab -e）
0 2 * * * cd /mnt/c/uep-v5-ultimate-enterprise-platform/projects/unified-platform && ./scripts/backup_db.sh ./backups


6. Cドライブの確保・Docker クリーンアップ

目安: 空き容量が 350GB を下回ったら実施。Docker ビルドを繰り返した後も推奨。

■ 軽いクリーンアップ（ビルドキャッシュのみ）
WSL Ubuntu ターミナルで実行:
docker builder prune -f

■ 本格的なクリーンアップ（未使用イメージ・コンテナ・ビルドキャッシュ）
docker system prune -a -f

■ ボリュームも削除する場合（データベース等のデータが消える）
docker system prune -a -f --volumes

■ Cドライブに容量を反映させる（VHD 圧縮）
1) PowerShell で WSL をシャットダウン:
   wsl --shutdown

2) 管理者 PowerShell で diskpart を起動し、以下を入力:
   select vdisk file="C:\Users\<ユーザー名>\AppData\Local\wsl\{56ae73c0-1bb0-4f94-8e60-27b79dbdb80f}\ext4.vhdx"
   compact vdisk
   exit

※ vhdx のパスは環境により異なる。実際のパスは以下で確認:
   Get-ChildItem -Path $env:LOCALAPPDATA -Recurse -Filter "ext4.vhdx" -ErrorAction SilentlyContinue | Select-Object FullName
