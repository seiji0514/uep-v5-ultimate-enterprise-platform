# トラブルシューティングガイド

## 🔧 よくある問題と解決方法

### 1. 既存システムが見つからない警告

**症状**:
```
医療サービスが見つかりませんでした。フォールバックモードで動作します。
法務サービスが見つかりませんでした。フォールバックモードで動作します。
金融サービスが見つかりませんでした。フォールバックモードで動作します。
```

**原因**:
- 既存システムのパスが正しく解決されていない
- 既存システムが存在しない、または別の場所にある

**解決方法**:

1. **既存システムの存在確認**:
   ```bash
   # ワークスペースルートで確認
   dir /b | findstr /i "次世代"
   dir /b | findstr /i "contract"
   ```

2. **パスの確認**:
   - 医療システム: `次世代マルチモーダルAI統合プラットフォームv8.0/app/services/healthcare_ai.py`
   - 金融システム: `次世代マルチモーダルAI統合プラットフォームv8.0/app/services/fintech_ai.py`
   - 法務システム: `contract-review-system/` ディレクトリ

3. **フォールバックモード**:
   - 既存システムが見つからない場合でも、フォールバックモードで動作します
   - 基本的な機能は利用可能ですが、高度な機能は制限されます

### 2. ポートが使用中エラー

**症状**:
```
ERROR: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8001): 
[winerror 10048] 通常、各ソケットアドレスに対してプロトコル、ネットワーク アドレス、またはポートのどれか1つのみを使用できます。
```

**原因**:
- ポート8001が既に別のプロセスで使用されている

**解決方法**:

1. **自動ポート検出（推奨）**:
   - システムが自動的に使用可能なポートを検出します
   - ログに表示されたポート番号を使用してください

2. **手動でポートを変更**:
   ```bash
   # .envファイルを作成
   SERVER_PORT=8002
   ```

3. **使用中のポートを確認**:
   ```bash
   # Windows
   netstat -ano | findstr :8001
   
   # プロセスを終了する場合
   taskkill /PID <PID番号> /F
   ```

### 3. 音声応答サービスが見つからない

**症状**:
```
既存の音声応答サービスが見つかりません。簡易実装を使用します。
```

**原因**:
- `ai-voice-response-system/backend/services/voice_response_service.py` が見つからない

**解決方法**:

1. **パスの確認**:
   - `ai-voice-response-system/backend/services/voice_response_service.py` が存在するか確認

2. **簡易実装**:
   - 簡易実装でも基本的な機能は動作します
   - 完全な機能を使用するには、既存の音声応答サービスが必要です

### 4. モジュールのインポートエラー

**症状**:
```
ModuleNotFoundError: No module named 'xxx'
```

**解決方法**:

1. **依存パッケージのインストール**:
   ```bash
   pip install -r requirements.txt
   ```

2. **仮想環境の確認**:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

### 5. WebSocket接続エラー

**症状**:
- ブラウザで接続できない
- WebSocket接続が確立されない

**解決方法**:

1. **ポート番号の確認**:
   - ブラウザのURLが正しいポート番号を使用しているか確認
   - ログに表示されたポート番号を使用

2. **ファイアウォールの確認**:
   - Windowsファイアウォールがポートをブロックしていないか確認

3. **ブラウザのコンソール確認**:
   - F12キーで開発者ツールを開く
   - Consoleタブでエラーメッセージを確認

## 📝 ログの確認方法

### ログレベル

- **INFO**: 通常の動作情報
- **WARNING**: 警告（動作は継続）
- **ERROR**: エラー（機能が制限される可能性）

### 重要なログメッセージ

1. **起動成功**:
   ```
   完全統合AI音声応答プラットフォームを起動します: http://0.0.0.0:8001
   ```

2. **ポート変更**:
   ```
   ポート8001が使用中のため、ポート8002を使用します。
   ```

3. **モジュール初期化**:
   ```
   医療サービスを初期化しました
   分野管理サービスを初期化しました: 6分野
   ```

## 🔍 デバッグ方法

### 1. 詳細ログの有効化

`backend/config.py` でログレベルを変更:
```python
LOG_LEVEL = "DEBUG"  # INFO, DEBUG, WARNING, ERROR
```

### 2. パスの確認

各モジュールでパスを確認:
```python
import logging
logger = logging.getLogger(__name__)
logger.info(f"PROJECT_ROOT: {PROJECT_ROOT}")
logger.info(f"医療サービスパス: {healthcare_path}")
```

### 3. ポートの確認

起動時にポートを確認:
```python
logger.info(f"使用ポート: {server_port}")
```
