# 面談情報設定ガイド

## 概要

面談自動応答システムで、**先方（企業・面接官）の情報**と**自分の情報**を設定することで、より精度の高い応答を生成できます。

## 設定できる情報

### 1. 自分の情報（応募者情報）

```json
{
  "name": "山田太郎",
  "experience": "5年間のエンジニア経験。AI技術を活用したWebアプリケーション開発に従事。",
  "skills": ["Python", "FastAPI", "AI", "機械学習", "Web開発"],
  "motivation": "AI技術を活用したサービス開発に貢献したいと考えています。",
  "strengths": "問題解決能力が高く、チームでの協調性も備えています。",
  "weaknesses": "完璧主義の傾向がありますが、優先順位をつけて効率的に進めるよう心がけています。"
}
```

### 2. 企業（先方）の情報

```json
{
  "name": "AIexe株式会社",
  "industry": "AI・機械学習",
  "business": "AI技術を活用したWebアプリケーション開発",
  "values": "技術革新とチームワークを重視",
  "culture": "フルリモート、裁量労働制、柔軟な働き方",
  "benefits": "健康診断、書籍購入補助、技術勉強会"
}
```

### 3. 面接官の情報

```json
{
  "name": "中山様",
  "position": "代表取締役",
  "department": "経営",
  "background": "AI技術の専門家。10年以上のエンジニア経験を持つ。"
}
```

## 設定方法

### 方法1: APIエンドポイントを使用

```bash
POST http://localhost:8001/api/interview/set-info
Content-Type: application/json

{
  "candidate_info": {
    "name": "山田太郎",
    "experience": "5年間のエンジニア経験",
    "skills": ["Python", "FastAPI", "AI"],
    "motivation": "AI技術を活用したサービス開発に貢献したい"
  },
  "company_info": {
    "name": "AIexe株式会社",
    "industry": "AI・機械学習",
    "business": "AI技術を活用したWebアプリケーション開発",
    "values": "技術革新とチームワーク"
  },
  "interviewer_info": {
    "name": "中山様",
    "position": "代表取締役",
    "department": "経営"
  }
}
```

### 方法2: フロントエンドから設定（今後実装予定）

UIから直接情報を入力できるフォームを追加予定です。

## 情報の活用方法

### 1. 志望動機の応答

**設定前**:
```
「なぜ当社を志望されましたか？」
→ 「貴社の[具体的な理由]に共感し、[自分のスキル]を活かして貢献したいと考えています。」
```

**設定後**:
```
「なぜ当社を志望されましたか？」
→ 「AIexe株式会社のAI技術を活用したサービス開発という事業内容に共感し、
   私のPythonやFastAPI、AI技術のスキルを活かして貢献したいと考えています。
   また、フルリモートや裁量労働制という柔軟な働き方にも魅力を感じています。」
```

### 2. 強みの応答

**設定前**:
```
「あなたの強みは何ですか？」
→ 「私の強みは[スキル]です。」
```

**設定後**:
```
「あなたの強みは何ですか？」
→ 「私の強みは問題解決能力とチームでの協調性です。
   5年間のエンジニア経験の中で、AI技術を活用したWebアプリケーション開発に
   従事してきた経験を活かせます。」
```

### 3. 企業文化への共感

**設定後**:
```
「当社の働き方についてどう思いますか？」
→ 「フルリモートと裁量労働制という柔軟な働き方は、私の働き方の価値観と
   非常に合致しています。効率的に成果を出せる環境で、自分のスキルを
   最大限に発揮できると考えています。」
```

## 情報設定のベストプラクティス

### 1. 具体的な情報を入力

- ❌ 曖昧: "エンジニア経験がある"
- ✅ 具体的: "5年間のエンジニア経験。AI技術を活用したWebアプリケーション開発に従事。"

### 2. 企業情報を事前に調べる

- 企業のWebサイト
- 採用情報ページ
- 企業のSNSやブログ
- 面接官のLinkedInプロフィール

### 3. 関連性を意識

- 自分のスキルと企業の事業内容の関連性
- 自分の価値観と企業の理念の一致点
- 自分の経験と企業のニーズのマッチング

## 使用例

### シナリオ1: 面談準備

1. 面談前に企業情報と面接官情報を設定
2. 自分の情報を入力
3. よくある質問を練習
4. AIが企業情報を考慮した応答を生成

### シナリオ2: リアルタイム支援

1. 面談中にシステムを起動
2. 質問を音声認識
3. AIが先方情報と自分の情報を考慮して応答を生成
4. 画面に表示された応答を確認してから回答

## 注意事項

1. **情報の正確性**: 設定する情報は正確なものを使用してください
2. **プライバシー**: 個人情報や機密情報は適切に管理してください
3. **透明性**: 実際の面談でAI支援を使用する場合は、相手に伝えることを推奨します

## APIリファレンス

### 情報設定

```http
POST /api/interview/set-info
Content-Type: application/json

{
  "candidate_info": {...},
  "company_info": {...},
  "interviewer_info": {...}
}
```

### 情報取得

```http
GET /api/interview/get-info
```

### 面談モードの切り替え

WebSocket経由で制御メッセージを送信：

```json
{
  "type": "set_interview_mode",
  "enabled": true
}
```

## まとめ

先方の情報と自分の情報を設定することで、**より精度の高い、企業に合わせた応答**を生成できます。

面談準備やリアルタイム支援に活用してください。
