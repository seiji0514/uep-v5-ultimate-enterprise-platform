# 本番デプロイ前 必須設定ガイド

**※ 代行設定済み**: `backend/.env` と `frontend/.env.production` を作成済み。  
**ローカル確認**: 現設定で本番ビルドのローカル確認が可能。  
**本番デプロイ時**: `python scripts/set-production-domain.py your-domain.com` でドメインを一括設定。

## 1. SECRET_KEY を本番用のランダム文字列に変更

```bash
# 64バイトのランダムキーを生成
python scripts/generate-secret-key.py
```

生成された文字列を `.env` または環境変数に設定:

```
SECRET_KEY=<生成された文字列>
```

---

## 2. CORS_ORIGINS に本番フロントのURLを設定

バックエンドの `.env` に追加:

```
# 単一オリジン
CORS_ORIGINS=https://app.your-domain.com

# 複数オリジン（カンマ区切り）
CORS_ORIGINS=https://app.your-domain.com,https://admin.your-domain.com
```

---

## 3. REACT_APP_API_URL に本番バックエンドURLを設定

フロントエンドの `.env.production` またはビルド時の環境変数:

```
REACT_APP_API_URL=https://api.your-domain.com
```

※ `npm run build` 実行時にこの値が埋め込まれます。

---

## 4. REACT_APP_SENTRY_DSN を設定（任意・推奨）

1. [sentry.io](https://sentry.io) でプロジェクト作成
2. DSN を取得
3. フロントエンドの `.env.production` に追加:

```
REACT_APP_SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
```

---

## 設定例（.env.production）

```env
# フロントエンド
REACT_APP_API_URL=https://api.example.com
REACT_APP_ENV=production
REACT_APP_SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
```

```env
# バックエンド
ENVIRONMENT=production
SECRET_KEY=<generate-secret-key.py で生成>
CORS_ORIGINS=https://app.example.com
DATABASE_URL=postgresql://user:pass@host:5432/dbname
```
